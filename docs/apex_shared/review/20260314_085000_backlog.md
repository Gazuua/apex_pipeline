## Critical

### 1. PendingCommand 스택 dangling 포인터 (UAF)

| 항목 | 내용 |
|------|------|
| **위치** | `apex_shared/lib/adapters/redis/src/redis_multiplexer.cpp:36-76` |
| **발견자** | reviewer-concurrency (Confidence 90), reviewer-logic (Confidence 75, FIFO 매칭 관점) |
| **현재 상태** | 코드에 TODO 주석 존재, v0.5에서 heap 할당 전환 예정 |
| **연관 이슈** | Finding 2 (on_disconnect pending 정리, 같은 파일 119-129행) |
| **권장 해결 시점** | v0.5 |

**수정 불가 사유**: PendingCommand 수명 관리 전체 재설계 필요. `command`, `static_on_reply`, `on_disconnect`, `close`, 소멸자가 연쇄적으로 PendingCommand에 의존하므로, 스택 할당에서 heap 할당으로의 전환은 단일 파일 수정으로 불가능. v0.5 마일스톤에서 Redis 멀티플렉서 재설계와 함께 처리 예정.

## Important

### 3. Redis AUTH 미구현 + password 필드 미사용

| 항목 | 내용 |
|------|------|
| **위치** | `apex_shared/lib/adapters/redis/src/redis_adapter.cpp:23` |
| **발견자** | reviewer-security (Important) |
| **현재 상태** | `RedisConfig::password` 필드 존재하나 연결 시 AUTH 명령 미전송 |
| **관련 코드** | `redis_adapter.cpp`, `redis_connection.hpp` |
| **권장 해결 시점** | v0.5 |

**수정 불가 사유**: Redis AUTH 처리 추가는 `RedisConnection` 연결 플로우 전체 변경 필요. 연결 성립 직후 AUTH → (optional) SELECT 시퀀스를 삽입해야 하며, 이는 비동기 연결 핸드셰이크 재설계에 해당. v0.5에서 Redis 어댑터 개편 시 함께 구현 예정.
