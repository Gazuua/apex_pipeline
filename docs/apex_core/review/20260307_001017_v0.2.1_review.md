## 1차 리뷰 발견 이슈 (11건)

### Critical (2건)

**C-NEW-1. server.cpp read_loop — writable() buffer overrun**
- **파일:** `src/server.cpp:107-110`
- `writable()`은 연속 영역만 반환하므로 wrap-around 시 `w.size() < n` 가능
- `writable_size() >= n` 통과 후 `memcpy(w.data(), tmp_buf.data(), n)`에서 경계 초과
- **수정:** 분할 복사 로직 추가 (`w.size() >= n` 분기 + else 분할)
- **2차 리뷰 검증:** OK

**C-NEW-2. Session async_send — 동시 쓰기 보호 문서화 필요**
- **파일:** `include/apex/core/session.hpp:39`
- 단일 io_context implicit strand에서는 안전하지만 명시적 문서화 필요
- **수정:** `@pre implicit strand에서 호출` 주석 추가
- **2차 리뷰 검증:** OK

### Important (4건)

**I-NEW-1. TimingWheel overflow 검사 누락 (I-3 불완전)**
- **파일:** `src/timing_wheel.cpp:9-15`
- MpscQueue, RingBuffer는 수정되었으나 TimingWheel 생성자 누락
- **수정:** `if (num_slots_ == 0) throw std::overflow_error(...)` 추가
- **2차 리뷰 검증:** OK

**I-NEW-2. server.cpp on_accept — 중복 set_state**
- **파일:** `src/server.cpp:84`
- `create_session` 내부에서 이미 `Active`로 전이, on_accept에서 중복 호출
- **수정:** 중복 호출 제거 + 주석 추가
- **2차 리뷰 검증:** OK

**I-NEW-3. test_service_base.cpp — 미사용 static io_context**
- **파일:** `tests/unit/test_service_base.cpp:77-80`
- `get_io_ctx()` 함수가 어디서도 호출되지 않는 데드코드
- **수정:** 함수 제거
- **2차 리뷰 검증:** OK

**I-NEW-4. test_server_e2e.cpp stop_and_join — 이중 post**
- **파일:** `tests/integration/test_server_e2e.cpp:171-175`

- **수정:** `server.stop()` 직접 호출로 변경
- **2차 리뷰 검증:** OK

## 코루틴 수명 안전성 평가

| 컴포넌트 | 수명 보장 메커니즘 | 평가 |
|----------|-------------------|------|
| `read_loop(SessionPtr)` | SessionPtr 값 복사 → 코루틴 프레임 소유 | 안전 |
| `process_frames(SessionPtr)` | SessionPtr 값 복사 → 코루틴 프레임 소유 | 안전 |
| `accept_loop()` | TcpAcceptor 멤버 함수, Server가 소유, stop()에서 acceptor close | 안전 |
| `handle/route` 람다 | `self` raw pointer, 서비스 수명 = Server 수명 | 조건부 안전 |
| `co_spawn(..., detached)` | 코루틴 내 예외를 catch하여 누수 없음 | 적절 |

## 동시성 안전 평가

단일 io_context + 단일 스레드 모델에서 모든 코루틴이 협력적 멀티태스킹. Boost.Asio implicit strand가 동작하여 동시성 문제 없음. 향후 multi-core (io_context-per-core) 확장 시 cross-core send 보호 필요.

## 핵심 결론

v0.2.0에서 가장 임팩트가 컸던 **C-1 (동기 블로킹 write)** 가 코루틴 전환으로 근본적으로 해결되었다. `co_await async_write` + `as_tuple(use_awaitable)` 패턴으로 io_context 스레드 블로킹이 완전히 제거되었으며, 핸들러까지 코루틴으로 통일하여 동기/비동기 혼용 문제가 사라졌다.

1차 리뷰에서 발견된 **writable() buffer overrun**은 실제 크래시를 유발할 수 있는 버그였으며, 분할 복사 로직으로 수정 완료. 2차 리뷰에서 이슈 0건으로 확인되어 **머지 가능 상태**로 판단한다.
