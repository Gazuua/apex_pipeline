## Issues

### Important (Should Fix) — 3건

#### I-1. config.cpp int64_t → uint16_t 나로잉 범위 검증 없음
- **파일**: `apex_core/src/config.cpp:29`
- **문제**: `port = 99999` → `static_cast<uint16_t>(99999)` = 34463, 에러 없이 잘림
- **영향**: 잘못된 포트/코어 수로 서버 동작
- **수정**: 범위 체크 후 `std::invalid_argument` throw

#### I-2. examples spdlog::get("app") null check 없음
- **파일**: `echo_server.cpp:85`, `multicore_echo_server.cpp:87`, `chat_server.cpp:97`
- **문제**: server.cpp는 null-safe 패턴 사용하나 예제는 미적용 — 일관성 부재
- **수정**: `if (auto app = spdlog::get("app")) app->info(...)` 패턴 적용

#### I-3. 빈 TOML 파일 테스트 부재
- **파일**: `apex_core/tests/unit/test_config.cpp`
- **문제**: 0바이트 TOML 파일에 대한 명시적 테스트 없음
- **수정**: `write_toml("empty.toml", "")` → 기본값 반환 검증 테스트 추가

### Minor (Nice to Have) — 5건

#### M-1. JsonFormatter 제어 문자 이스케이핑 불완전
- **파일**: `apex_core/src/logging.cpp:44-52`
- **문제**: `\b`, `\f`, U+0000~U+001F 범위 미처리 (RFC 8259)
- **수정**: 0x00~0x1F 범위를 `\uXXXX` 로 이스케이프

#### M-2. init_logging/shutdown_logging 비대칭 주석 부재
- **파일**: `apex_core/src/logging.cpp:68, 107`
- **문제**: shutdown 후 재초기화 안전성에 대한 코멘트 없음
- **수정**: `shutdown_logging()` 또는 `init_logging()` 에 안전성 주석 추가

#### M-3. CI GCC 버전 미지정
- **파일**: `.github/workflows/ci.yml`
- **문제**: `ubuntu-latest` 기본 GCC가 C++23 미완전 지원 가능성
- **수정**: `g++-14` 명시 설치 또는 `setup-cpp` 액션 사용

#### M-4. default.toml num_cores = 0 vs ServerConfig 기본값 = 1 주석 부재
- **파일**: `apex_core/config/default.toml:6`
- **문제**: 혼동 여지
- **수정**: 주석으로 차이 명시

#### M-5. 설계서 테스트 파일명 불일치
- **파일**: `docs/apex_core/plans/20260308_070528_phase5-design.md:297-299`
