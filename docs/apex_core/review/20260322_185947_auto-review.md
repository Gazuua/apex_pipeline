## 수정 내역

### 1. RedisMultiplexer::connect() IIFE 람다 코루틴 → initial_auth() 멤버 함수

- **파일**: `redis_multiplexer.hpp`, `redis_multiplexer.cpp`
- **증상**: E2E 테스트 segfault (SIGSEGV)
- **원인**: `connect()` 내부에서 즉시 호출된 람다(IIFE)를 코루틴으로 사용. 람다 임시 객체가 문장 끝에서 파괴되지만, 코루틴 프레임이 파괴된 람다의 `this`를 계속 참조 → dangling this → SIGSEGV.
- **수정**: IIFE 람다 코루틴을 `initial_auth()` 멤버 함수로 추출하여 코루틴 프레임의 수명이 객체 수명에 안전하게 묶이도록 변경.

### 2. Server::~Server() io_context restart+poll 추가

- **파일**: `server.cpp`
- **증상**: ASAN UAF (test_server_e2e)
- **원인**: `io_context::stop()` 호출 후 미처리 timer cancel completion handler가 남아 있는 상태에서 `~io_context()`가 코루틴 프레임을 강제 파괴. 이후 timer_service가 이미 파괴된 코루틴 프레임에 재진입하여 use-after-free 발생.
- **수정**: `io_context::restart()` + `io_context::poll()`을 `listeners_.clear()` 이전에 배치. 모든 객체가 유효한 상태에서 미처리 completion handler를 소진시켜 UAF 방지.

### 3. spawn_adapter_coro() 내부를 post()로 래핑

- **파일**: `adapter_base.hpp`
- **증상**: CancellationToken 스레드 assert 실패
- **원인**: `spawn_adapter_coro()`가 main 스레드에서 `new_slot()` 호출 → 코루틴 완료 시 core 스레드에서 `on_complete()` 실행 → CancellationToken의 owner thread assert 실패 (생성 스레드 ≠ 완료 스레드).
- **수정**: `spawn_adapter_coro()` 내부 로직을 `post()`로 래핑하여 `new_slot()` 호출이 올바른 executor 스레드에서 실행되도록 보장.
