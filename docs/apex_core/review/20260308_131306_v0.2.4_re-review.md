## Important — 8건

### I-1. TimingWheel tick() free_ids_ 불변식 주석 누락

- **파일**: `src/timing_wheel.cpp` Phase 3b 루프
- **신뢰도**: 75%
- **내용**: 콜백 호출 전에 free_ids_.push_back을 하지 않는 순서가 재진입 안전성을 보장하지만, 이 불변식이 주석으로 명시되어 있지 않음
- **수정**: Phase 3b 루프에 불변식 주석 추가

### I-2. MpscQueue 32비트 빌드 방어 미비

- **파일**: `include/apex/core/mpsc_queue.hpp`
- **신뢰도**: 40%
- **내용**: size_t 단조 증가 인덱스가 32비트에서 wrap-around 가능. 현재 64비트 전용이나 명시적 가드 없음
- **수정**: `static_assert(sizeof(size_t) >= 8, ...)` 추가

### I-3. TcpAcceptor accept_loop context_provider 분기 코드 중복

- **파일**: `src/tcp_acceptor.cpp:46-81`
- **신뢰도**: 90%
- **내용**: context_provider_ 유무에 따른 if/else 분기에서 에러 핸들링 + 콜백 호출 코드가 완전 동일하게 중복
- **수정**: 소켓 생성 부분만 분기, 에러 핸들링/콜백은 공통화

### I-4. Session::close() set_state() 우회 의도 미주석

- **파일**: `src/session.cpp:62-70`
- **신뢰도**: 95%
- **내용**: close()가 set_state()를 우회하여 직접 state_ = Closed 대입. 의도적이나 주석 없음
- **수정**: 주석 한 줄 추가

### I-5. TcpAcceptor StopPreventsNewAccepts 50ms sleep 잔존

- **파일**: `tests/unit/test_tcp_acceptor.cpp:95`
- **신뢰도**: 85%
- **내용**: stop 후 50ms sleep으로 accept_count==0 검증 — CI 환경에서 타이밍 의존적
- **수정**: connect 시도 자체의 에러 반환 확인 방식으로 전환

### I-6. CrossCoreCallQueueFullTest drain_remaining 메모리 누수 미검증

- **파일**: `tests/unit/test_cross_core_call.cpp:141-187`
- **신뢰도**: 75%
- **내용**: queue에 남은 task의 정리가 drain_remaining에 의존하나 명시적 누수 검증 없음
- **수정**: 테스트 주석에 ASan 빌드 검증 안내 추가 + 카운터 기반 정리 확인

### I-7. Apex_Pipeline.md io_context-per-core 소켓 이관 "미완" — 실제 구현 완료
