# CI 긴급 수정 — E2E segfault + ASAN UAF + CancellationToken assert

**날짜**: 2026-03-22
**브랜치**: bugfix/ci-uaf-fixes
**PR**: #110

## 배경

main 브랜치 CI에서 E2E segfault, ASAN UAF, CancellationToken 스레드 assert 3건이 동시 발생. CI 정상화를 위한 긴급 수정.

## 변경 파일

| 파일 | 변경 내용 |
|------|-----------|
| `apex_core/src/server.cpp` | ~Server() 소멸자에 io_context restart+poll 추가 (ASAN UAF 수정) |
| `apex_shared/include/apex/adapters/adapter_base.hpp` | spawn_adapter_coro() 내부를 post()로 래핑 (CancellationToken 스레드 assert 수정) |
| `apex_shared/include/apex/adapters/redis_multiplexer.hpp` | IIFE 람다 코루틴을 initial_auth() 멤버 함수로 추출 (선언부) |
| `apex_shared/src/adapters/redis_multiplexer.cpp` | initial_auth() 멤버 함수 구현 (E2E segfault 수정) |

총 변경: +56/-28

## 커밋 요약

1. **fix(shared): RedisMultiplexer IIFE 람다 코루틴 → initial_auth() 멤버 함수 추출** — 즉시 호출 람다 임시객체 파괴 후 코루틴 프레임이 dangling this를 참조하여 발생한 SIGSEGV 수정.
2. **fix(core): Server::~Server() io_context restart+poll로 미처리 핸들러 소진** — io_context::stop() 후 미처리 timer cancel completion handler가 코루틴 프레임 강제 파괴를 유발한 ASAN UAF 수정.
3. **fix(shared): spawn_adapter_coro() post() 래핑으로 스레드 소유권 보장** — main 스레드에서 new_slot(), core 스레드에서 on_complete() 실행되며 owner thread assert 실패하던 문제 수정.

## 영향

- main 브랜치 CI 정상화: E2E segfault 해소, ASAN test_server_e2e UAF 해소
- CancellationToken 스레드 assert 안정화로 어댑터 코루틴 스폰 안전성 확보
