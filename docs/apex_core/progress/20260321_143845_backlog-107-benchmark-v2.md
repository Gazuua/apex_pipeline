# BACKLOG-107: 벤치마크 보고서 v2 — 완료 기록

## 결과 요약

벤치마크 보고서 시스템을 Release vs Debug 비교에서 **버전 간 비교 + 7개 방법론 비교** 체계로 전면 재설계.
PDF에서 HTML(Plotly 인터랙티브 차트, 다크 대시보드 테마)로 전환.

## 주요 변경

### 벤치마크 (C++)
- `bench_allocators.cpp` — Slab + Bump + Arena + malloc + make_shared (bench_slab_allocator 대체)
- `bench_serialization.cpp` — FlatBuffers Build/Read vs HeapAlloc Build/Read
- `bench_architecture_comparison.cpp` — Per-core vs Shared(64-shard) stateful 워크로드
- `bench_dispatcher.cpp` — SessionManager 스케일 flat_map vs std::unordered_map (lookup + iteration)
- `bench_cross_core_latency.cpp` — tick_interval 100ms→1ms + semaphore 동기화
- `bench_session_lifecycle.cpp` — shared_ptr 비교 추가
- `bench_ring_buffer.cpp` — naive memcpy 비교 추가

### 보고서 시스템 (Python)
- `generate_benchmark_report.py` — HTML 출력 (Plotly + 다크 테마), --baseline/--current CLI
- `architecture_comparison.html` — Per-core 아키텍처 대문짝 showcase
- `architecture_scaling.png` — README 대문 차트 (정적 PNG)

### 핵심 벤치마크 결과

| 방법론 비교 | 결과 |
|------------|------|
| SPSC vs MPSC | SPSC 1P1C에서 일관 우위 |
| Bump vs malloc | 7ns vs 120ns (17x) |
| Slab vs malloc | 11ns vs 120ns (11x) |
| intrusive_ptr vs shared_ptr | 6.3ns vs 18ns (2.9x) |
| zero-copy vs naive memcpy | 35ns vs 156ns (4.5x) |
| FlatBuffers Read vs heap | 19~32x 빠름 (zero-copy) |
| flat_map iteration vs std | 26ms vs 66ms (2.5x, 10K entries) |
| Per-core vs Shared (4워커) | 1.56M vs 0.74M msg/s (2.1x) |

### Per-core 아키텍처 선형 확장 검증
- 1→2→3→4워커에서 0.51→0.90→1.24→1.56M msg/s (77~88% 효율)
- Shared 모델은 64-shard 최적화 적용에도 0.77M에서 정체
- 병목: io_context 내부 핸들러 큐의 단일 mutex
- 결론: io_context 분리가 선형 확장의 유일한 해법
