## 2. Critical 수정 (core 관련 4건)

### C-1. IOCP 소켓 바인딩 수정
- **문제**: accept_io_에서 생성된 소켓이 core별 io_context로 이관되지 않아 멀티코어 분산 무효화
- **수정**: TcpAcceptor에 `ContextProvider` 콜백 추가, `async_accept(target_ctx, token)` 사용으로 소켓을 처음부터 target core IOCP에 바인딩
- **파일**: `tcp_acceptor.hpp/cpp`, `server.cpp`

### C-2. process_frames payload 안전성
- **문제**: `co_await dispatcher.dispatch()` 중 RingBuffer 내부를 가리키는 payload span 무효화 가능
- **수정**: `consume_frame()`을 dispatch 전 호출, payload를 로컬 vector로 복사
- **파일**: `server.cpp`

### C-3. FrameError 미처리 → 무한 파싱 루프
- **문제**: `try_decode`의 `BodyTooLarge`/`HeaderParseError` 시 세션 close 없이 버퍼에 악성 데이터 잔존
- **수정**: 에러 종류별 분기 — `InsufficientData`면 break, 그 외면 `session->close()` + `co_return`
- **파일**: `server.cpp`

### C-4. RingBuffer linearize() span 수명 미문서화
- **문제**: contiguous→buffer 포인터, wrap→linear_buf_ 포인터 — consume/다음 linearize 시 댕글링
- **수정**: `@warning` 주석 추가 + `write()` 편의 메서드 추가로 수동 split-write 제거
- **파일**: `ring_buffer.hpp/cpp`
