## 주요 설계 결정

### Server::run() 블로킹 모델
- 프레임워크가 io_context + 스레드 전부 소유 (Seastar/Actix 모델)
- `Server(io_ctx&)` 생성자 제거, 하위 호환 불필요

### Windows IOCP Graceful Shutdown
- `async_accept`로 받은 소켓은 accept_io_의 IOCP에 영구 바인딩
- accept_io_를 shutdown 중에도 살려둬야 세션 소켓의 IOCP 완료 처리 가능
- begin_shutdown() → poll_shutdown() 비동기 패턴으로 해결
- work_guard로 accept_io_ 유지, 세션 0 도달 시 core_engine stop → accept_io_ stop

### cross_core_call CAS 패턴
- shared State의 atomic<int> status로 완료/타임아웃 경쟁 해결
- status: 0=pending, 1=done, 2=timeout
- 결과를 status CAS 전에 기록 (seq_cst 보장)
- shared_ptr<steady_timer>로 크로스 스레드 cancel 안전성 확보

### CoreEngine drain loop 자동 실행
- CrossCoreRequest/Post 메시지의 data 필드에 std::function<void()>* 전달
- drain loop에서 자동 실행 + delete (MpscQueue trivially_copyable 제약 유지)
