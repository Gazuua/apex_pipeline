# v0.2.4 Server-CoreEngine 통합 — 설계 완료 체크포인트

## 완료 사항

### 브레인스토밍 (설계 결정 확정)
- D-1: Server::run() 블로킹, 프레임워크가 전부 소유 (Seastar 모델)
- D-1 후속: 기본 시그널 처리 + thread-safe stop() (Actix 패턴)
- D-1 하위 호환: 불필요, 전면 재설계
- ServiceBase: 코어별 독립 인스턴스 (Seastar sharded<T>)
- D-2: raw new/delete unique ownership, 단일 큐, API 래퍼
- D-3: Redis Pub/Sub 브로드캐스트, cross_core_post는 프레임워크 내부 전용
- C-1: async_accept(target_executor) 소켓 이관
- I-1: drain_timer 콜백에서 tick 통합
- I-2: steady_timer cancel 패턴 resume
- I-4: 5단계 graceful shutdown 시퀀스

### 설계 초안 분석 — 발견된 이슈
- Critical 3건 (소켓 executor, 포인터 수명, CoreEngine 블로킹) → 전부 설계에 반영
- Important 6건 (tick, resume, ServiceBase, shutdown 등) → 전부 설계에 반영
- Minor 3건 (주석, 에러코드, 테스트 타이밍) → 구현 계획에 반영
