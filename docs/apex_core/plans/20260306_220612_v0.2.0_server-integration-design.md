# Phase 4.5: Server 통합 + E2E 테스트 설계

> **목표:** 프레임워크 컴포넌트를 하나로 엮는 Server 클래스, 재사용 가능한 TcpAcceptor, 실질적인 E2E 테스트, 예제 2개 → v0.2.0

---

## 1. 핸들러 시그니처 변경

모든 핸들러에 `SessionPtr`을 첫 인자로 추가.

기존:
```cpp
using Handler = std::function<void(uint16_t, std::span<const uint8_t>)>;
void on_echo(uint16_t msg_id, std::span<const uint8_t> payload);
void on_echo(uint16_t msg_id, const EchoRequest* req);
```

변경:
```cpp
using Handler = std::function<void(SessionPtr, uint16_t, std::span<const uint8_t>)>;
void on_echo(SessionPtr session, uint16_t msg_id, std::span<const uint8_t> payload);
void on_echo(SessionPtr session, uint16_t msg_id, const EchoRequest* req);
```

세션 없이 디스패치하는 기존 테스트는 `nullptr` 전달.

**영향:** MessageDispatcher, ServiceBase, handle(), route<T>(), 기존 테스트 6개 수정.

## 2. TcpAcceptor

재사용 가능한 비동기 TCP accept 루프.

```cpp
class TcpAcceptor {
public:
    using AcceptCallback = std::function<void(tcp::socket)>;

    TcpAcceptor(boost::asio::io_context& io_ctx, uint16_t port,
                AcceptCallback on_accept);

    void start();
    void stop();
    uint16_t port() const noexcept;

private:
    void do_accept();

    tcp::acceptor acceptor_;
    AcceptCallback on_accept_;
    bool running_{false};
};
```

- io_context 외부 주입 -> CoreEngine과 결합 용이
- port=0 -> OS가 빈 포트 할당 (테스트용)
- 콜백 기반 -> Server가 콜백에서 세션 생성

## 3. ServiceBaseInterface

CRTP ServiceBase의 비템플릿 인터페이스. Server가 서비스를 소유하기 위해 필요.

```cpp
class ServiceBaseInterface {
public:
    virtual ~ServiceBaseInterface() = default;
    virtual void start() = 0;
    virtual void stop() = 0;
    virtual std::string_view name() const = 0;
};

template <typename Derived>
class ServiceBase : public ServiceBaseInterface { ... };
```

## 4. Server

모든 컴포넌트를 엮는 최상위 클래스.

```cpp
class Server {
public:
    struct Config {
        uint16_t port = 9000;
        uint32_t heartbeat_timeout_ticks = 300;
        uint32_t tick_interval_ms = 100;
        size_t recv_buf_capacity = 8192;
    };

    explicit Server(boost::asio::io_context& io_ctx, Config config = {});

    template <typename T>
    T& add_service();

    void start();
    void stop();

    SessionManager& session_manager() noexcept;
    MessageDispatcher& dispatcher() noexcept;
    uint16_t port() const noexcept;

private:
    void on_accept(tcp::socket socket);
    void start_read(SessionPtr session);
    void do_read(SessionPtr session);
    void process_frames(SessionPtr session);
    void start_tick_timer();

    boost::asio::io_context& io_ctx_;
    Config config_;
    TcpAcceptor acceptor_;
    SessionManager session_mgr_;
    MessageDispatcher dispatcher_;
    boost::asio::steady_timer tick_timer_;
    std::vector<std::unique_ptr<ServiceBaseInterface>> services_;
};
```

핵심 흐름:
1. on_accept() -> session_mgr_.create_session() -> start_read()
2. do_read() -> async_read_some -> RingBuffer -> process_frames()
3. process_frames() -> FrameCodec::try_decode -> dispatcher_.dispatch(session, msg_id, payload) -> touch_session()
4. start_tick_timer() -> steady_timer로 주기적 session_mgr_.tick()

## 6. 예제

### Echo Server (업그레이드)

기존 140줄 수동 코드 -> Server 기반 ~20줄:

```cpp
class EchoService : public ServiceBase<EchoService> {
    void on_start() override {
        route<EchoRequest>(0x0001, &EchoService::on_echo);
    }
    void on_echo(SessionPtr session, uint16_t msg_id, const EchoRequest* req) {
        // 응답 빌드 -> session->send()
    }
};

int main() {
    boost::asio::io_context io_ctx;
    Server server(io_ctx, {.port = 9000});
    server.add_service<EchoService>();
    server.start();
    io_ctx.run();
}
```

### Chat Server (신규)

크로스 세션 통신 시연:

```cpp
class ChatService : public ServiceBase<ChatService> {
    void on_start() override {
        route<ChatMessage>(0x0100, &ChatService::on_chat);
    }
    void on_chat(SessionPtr sender, uint16_t msg_id, const ChatMessage* msg) {
        auto frame = build_broadcast(sender->id(), msg);
        session_manager_->for_each([&](SessionPtr s) {
            s->send_raw(frame);
        });
    }
};
```

신규 스키마: `chat_message.fbs`

## 7. E2E 통합 테스트

파일: `tests/integration/test_server_e2e.cpp`

| 테스트 | 검증 |
|--------|------|
| ServerAcceptAndEcho | 연결 -> EchoRequest -> EchoResponse 수신 |
| MultipleClients | 3개 동시 접속, 독립 에코 |
| HeartbeatTimeoutDisconnect | 무응답 -> 타임아웃 -> 연결 끊김 |
| InvalidMessageErrorResponse | 미등록 msg_id -> ErrorResponse(HandlerNotFound) |
| InvalidFlatBufferErrorResponse | 깨진 페이로드 -> ErrorResponse(FlatBuffersVerifyFailed) |
| GracefulShutdown | server.stop() -> 모든 세션 정리 |
| ChatBroadcast | A 전송 -> A, B 모두 수신 |

테스트 패턴: port=0, io_ctx 별도 스레드, 클라이언트 소켓으로 직접 프레임 송수신.
