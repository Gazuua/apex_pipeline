**I-2. TimingWheel 힙 할당 — SessionManager에서 reschedule 활용**

`src/session_manager.cpp:59-71` `touch_session()` 수정:

현재:
```cpp
timer_wheel_.cancel(old_timer);  // delete entry
auto new_timer = timer_wheel_.schedule(heartbeat_timeout_ticks_);  // new entry
```

수정:
```cpp
timer_wheel_.reschedule(timer_it->second, heartbeat_timeout_ticks_);
// cancel+schedule 대신 reschedule로 Entry 재사용 — new/delete 제거
```

주의: `reschedule()`은 timer_to_session_ 매핑을 변경하지 않으므로 기존 매핑 유지 가능.
`touch_session()` 전체를 단순화:
```cpp
void SessionManager::touch_session(SessionId id) {
    if (heartbeat_timeout_ticks_ == 0) return;
    auto timer_it = session_to_timer_.find(id);
    if (timer_it == session_to_timer_.end()) return;
    timer_wheel_.reschedule(timer_it->second, heartbeat_timeout_ticks_);
}
```

추가: `reschedule()`에도 범위 검증 추가 (I-4와 연계).

**I-3. SlabPool Release double-free 보호**

`src/slab_pool.cpp` 수정:

```cpp
// 슬롯 상태 마커 (allocated/freed 구분)
static constexpr uint32_t SLAB_MAGIC_ALLOCATED = 0xA110CA7E;
static constexpr uint32_t SLAB_MAGIC_FREED     = 0xF2EED000;
```

- `allocate()`: 반환 전 슬롯 시작 4바이트에 `SLAB_MAGIC_ALLOCATED` 기록
  - 주의: slot_size_가 `sizeof(FreeNode)` + `sizeof(uint32_t)` 이상이어야 함
  - 또는 FreeNode 구조에 magic 필드 추가
- `deallocate()`: magic이 `ALLOCATED`인지 확인 → 아니면 early return (또는 로그)
  - `FREED`이면 double-free, 다른 값이면 외부 포인터
  - 확인 후 `SLAB_MAGIC_FREED`로 변경
- `owns()`: 슬롯 정렬 검증 추가 `(p - chunk.data) % slot_size_ == 0`

설계 결정: magic 검증 실패 시 throw vs silent return?
→ `noexcept` 시그니처 유지를 위해 **silent return + assert(false)** 조합 권장.
  Release에서는 return으로 corruption 방지, Debug에서는 assert로 즉시 포착.

**I-4. assert → 런타임 체크 (TimingWheel + RingBuffer)**

| 파일 | 현재 | 수정 |
|------|------|------|
| `src/timing_wheel.cpp:54` | `assert(ticks < num_slots_)` | `if (ticks >= num_slots_) throw std::out_of_range(...)` |
| `src/timing_wheel.cpp:96` (reschedule) | 범위 검증 없음 | 동일한 throw 추가 |
| `src/ring_buffer.cpp:70` (commit_write) | `assert(n <= writable_size())` | `n = std::min(n, writable_size())` (clamp, hot path) |
| `src/ring_buffer.cpp:76` (consume) | `assert(n <= readable_size())` | `n = std::min(n, readable_size())` (clamp, hot path) |

RingBuffer는 hot path이므로 throw 대신 clamp. TimingWheel schedule은 세션 생성 시 한 번이므로 throw 허용.

**I-6. TcpAcceptor accept_loop backoff**

`src/tcp_acceptor.cpp:40-48` 수정:

```cpp
boost::asio::awaitable<void> TcpAcceptor::accept_loop() {
    boost::asio::steady_timer backoff_timer(io_ctx_);

    while (running_.load(std::memory_order_relaxed)) {
        auto [ec, socket] = co_await acceptor_.async_accept(
            boost::asio::as_tuple(boost::asio::use_awaitable));

        if (ec) {
            if (ec == boost::asio::error::operation_aborted) break;
            // I-6: 일시적 에러 시 100ms 백오프 (EMFILE 등 busy-loop 방지)
            backoff_timer.expires_after(std::chrono::milliseconds(100));
            co_await backoff_timer.async_wait(boost::asio::use_awaitable);
            continue;
        }

        on_accept_(std::move(socket));
    }
}
```
