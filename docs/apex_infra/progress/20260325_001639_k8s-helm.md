### 핵심 아키텍처 결정

- **2-Release 배포**: `apex-infra`(인프라) + `apex-services`(앱) namespace 분리
- **expand_env 활용**: 기존 C++ `${VAR:-default}` 파서로 Secret 환경변수 직접 주입 (init-container 불필요)
- **하이브리드 인프라**: 로컬=Bitnami sub-chart, 프로덕션=외부 관리형 서비스 (values 토글)
- **Gateway TLS 비활성**: K8s Ingress가 TLS termination 담당

### 서비스 코드 변경
