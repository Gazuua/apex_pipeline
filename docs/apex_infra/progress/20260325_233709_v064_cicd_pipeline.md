## 작업 결과

### 3단계 검증 파이프라인 확립

기존 e2e(호스트 바이너리) + helm-validation(minikube) 사이에 docker-compose 스모크 테스트 단계를 추가하여 3단계 검증 체계를 완성했다.

| 단계 | 도구 | 검증 대상 |
|------|------|-----------|
| e2e | 호스트 바이너리 | 비즈니스 로직 (코드 정합성) |
| smoke-test | docker-compose | 컨테이너 환경 (이미지 빌드 결과물) |
| helm-validation | minikube | K8s 오케스트레이션 (배포 매니페스트) |

### CI/CD 파이프라인 변경 (.github/workflows/ci.yml)

- **bake-services PR 확장**: PR에서도 서비스 이미지 빌드 검증 (push 없이 로컬 로드)
- **smoke-test 잡 신규**: docker-compose.smoke.yml 기반 4개 서비스 기동 + health check
- **deploy 잡 신규**: Argo Rollouts canary 배포 (10→50→100%, minikube, main-only)
- **조건부 태깅**: sha-{commit} 항상, latest + main 태그는 main 브랜치만
- **scan/helm-validation 조건 보정**: PR에서 GHCR 이미지 부재 시 스킵

### Helm 차트 확장

- Argo Rollouts CRD 지원 (rollout.yaml 4개 서비스 추가)
- `_helpers.tpl` podTemplate 공통 추출 (Deployment/Rollout 공유)
- values.yaml에 `rollouts.enabled`, `rollouts.canary` 설정 추가
- values-prod.yaml에 프로덕션 canary 전략 프리셋

### Docker 인프라

- `docker-compose.smoke.yml` 신규 (gateway + auth + chat + log + infra)
- `service.Dockerfile` debug/release 빌드 preset 분기 (`BUILD_PRESET` ARG)
- `docker-bake.hcl` 태깅 전략 갱신 (sha + latest + main 조건부)
- `.dockerignore`에 `**/*.pem` 제외 추가

### E2E 설정

- `auth_svc_docker.toml`, `chat_svc_docker.toml`, `gateway_docker.toml` 추가 (docker-compose 환경용)
