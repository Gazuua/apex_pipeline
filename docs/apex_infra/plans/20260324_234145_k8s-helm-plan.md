# v0.6.3 K8s + Helm 차트 구현 계획

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Umbrella Helm 차트로 전체 Apex Pipeline 스택을 K8s(minikube)에 배포 가능하게 만든다.

**Architecture:** Umbrella chart(`apex-pipeline`) + library chart(`apex-common`) + 서비스별 sub-chart(gateway, auth-svc, chat-svc, log-svc) + 인프라(Bitnami Kafka/Redis/PostgreSQL + custom PgBouncer). 2-release 전략(apex-infra/apex-services namespace 분리). expand_env 기반 Secret 주입.

**Tech Stack:** Helm 3, Kubernetes 1.28+, nginx-ingress, Bitnami Helm charts, minikube

**Spec:** `docs/apex_infra/plans/20260324_230624_k8s-helm-design.md`

- [ ] **Step 1: local-setup.sh**

```bash
#!/usr/bin/env bash
# Copyright (c) 2026 Gazuua. All rights reserved. Licensed under the MIT License.
# Apex Pipeline — minikube 로컬 환경 원클릭 셋업
set -euo pipefail

SCRIPT_DIR="$(cd "$(dirname "$0")/.." && pwd)"
cd "$SCRIPT_DIR"

# 1. minikube 상태 확인 + 시작
minikube status > /dev/null 2>&1 || minikube start --memory=4096 --cpus=2

# 2. Ingress addon
minikube addons enable ingress

# 3. Helm dependencies
helm dependency update .

# 4. Infra release (apex-infra namespace)
```
