## 발견 이슈 및 수정

### 이슈 1: hook 바이패스 false positive (medium)
- **문제**: `isDaemonManagementCommand`가 `strings.Contains(command, "daemon start")`로 매칭하므로, `git commit -m "fix daemon startup"` 같은 커밋도 handoff 검증을 우회
- **수정**: `isGitCommit(command)` 체크를 선행하여 git commit 명령은 항상 바이패스 대상에서 제외
- **테스트**: 2개 케이스 추가 (`"fix daemon startup issue"`, `"daemon start bug"`)

### 이슈 2: maintenance lock stale 잔류 (minor)
- **문제**: `daemon stop`이 maintenance 파일을 먼저 쓰고 종료를 시도하는데, IPC도 PID Kill도 모두 실패하면 "daemon not running" 에러를 반환하면서 maintenance 파일이 남음 → auto-restart 영구 억제
- **수정**: 양쪽 모두 실패한 경우 (`readPID` 에러) maintenance 파일 정리

## 리뷰 안건 (이슈 없음)

- **context deadline 전파**: `sendWithAutoRestart`에서 retry 시 원래 context가 그대로 전달됨. daemon 시작(5초) + retry 시간을 감안하면 짧은 timeout에서 여유가 줄어들지만, 실사용 timeout(30초~30분)에서는 문제 없음
- **config 하위 호환**: 기존 config.toml의 `idle_timeout` 필드는 `BurntSushi/toml`이 unknown field를 무시하므로 파싱 에러 없음
- **versionOnce 재사용**: auto-restart 후 동일 `ipc.Client`의 `versionOnce`가 이미 소비됨. 하지만 동일 바이너리이므로 버전 불일치 없음, 문제 없음
