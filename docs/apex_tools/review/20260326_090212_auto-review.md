## 수정 이슈 상세

### CRITICAL (1건)

#### C-1. watchdog killed 플래그 이중 목적 버그
- **파일**: `internal/cli/queue_cmd.go`
- **문제**: `killed` atomic.Bool이 "watchdog 정지 신호"와 "kill 표시" 두 가지 목적으로 사용. 빌드 프로세스가 정상 실패(non-zero exit)해도 `runWithBuildLock`이 Wait() 후 `killed.Store(true)`를 설정하여 이후 항상 "KILLED (stale)" 메시지가 잘못 출력됨.
- **수정**: `stopped`(정지 신호)와 `killed`(watchdog kill 표시) 두 개의 `atomic.Bool`로 분리. `watchBuildLog`/`watchBuildLogWithTimeout` 시그니처 동시 변경, 테스트 업데이트.

### MAJOR (6건)

#### M-1, M-2. backlog migration v3 rows.Err() 미검사
- **파일**: `internal/modules/backlog/module.go`
- **문제**: scope 정규화와 resolution 정규화 각각에서 `rows` iteration 후 `rows.Err()` 미검사. I/O 에러 시 부분 정규화된 데이터로 진행하여 데이터 불일치 유발 가능.
- **수정**: 각 `rows.Close()` 직전에 `rows.Err()` 체크 추가.

#### M-3, M-4, M-5. queue manager RowsAffected() 에러 무시

- **문제**: `tryPromote`, `UpdatePID`, `Release` 3곳에서 `res.RowsAffected()` 에러를 `_`로 무시. 에러 시 n=0이 반환되어 lock 소유권 판정, promote 판정, release 성공 판정이 잘못될 수 있음.
- **수정**: 에러 체크 추가.

#### M-6. config WriteDefault MkdirAll 에러 무시

- **문제**: `os.MkdirAll` 에러를 무시하여 디렉토리 생성 실패 시 후속 `WriteFile`에서 불명확한 에러 발생.
- **수정**: `fmt.Errorf`로 래핑하여 반환.

#### M-7. DataDir() 환경변수 미설정 시 위험한 경로 반환
- **파일**: `internal/platform/paths.go`
- **문제**: Windows에서 `LOCALAPPDATA`가 비어있으면 `\apex-agent`(루트 디렉토리)에, Unix에서 `UserHomeDir` 실패 시 상대경로에 DB/PID/로그가 생성됨.
- **수정**: 모든 환경변수 fallback 체인에서 에러 검사. Windows는 `UserHomeDir()`으로 `AppData/Local` 경로 재구성.

#### M-8. E2E testenv Restart() HTTP 서버 미설정
- **파일**: `e2e/testenv/env.go`
- **문제**: `Restart()` 메서드에서 HTTP 서버 팩토리 미설정, HTTP config 누락, HTTP 준비 대기 누락. 재시작 후 HTTP 관련 E2E 테스트가 HTTP 엔드포인트에 접근 불가.

### 설계 개선 (DESIGN_DECISION → 수정 완료)

#### D-1. MergeFullParams ImportFn/ExportFn context 캡처 불일치
- **파일**: `workflow/pipeline.go`, `handoff/module.go`, `workflow/pipeline_test.go`
- **문제**: `ImportFn`/`ExportFn` 클로저가 handler의 request context를 캡처. 다른 함수들(`PreMergeCheckFn`, `FinalizeFn`, `LockAcquireFn`, `LockReleaseFn`)은 pipeline이 인자로 전달하는 context를 사용. 현재 동작에 버그는 없으나, pipeline이 다른 context를 사용하게 되면 cancellation/timeout 전파가 깨짐.
- **수정**: `ImportFn`/`ExportFn` 시그니처에 `ctx context.Context` 추가, pipeline이 자체 context를 전달. `buildMergeParams`에서 불필요해진 `ctx` 파라미터 제거.
