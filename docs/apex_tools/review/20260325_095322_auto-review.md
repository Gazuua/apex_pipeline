## 리뷰 관점

### Logic
- `autoExport()`: non-fatal 설계 (mutation 성공 후 export 실패 시 warning만 출력). 적절함
- stale guard: `existing.UpdatedAt > importUpdatedAt` 문자열 비교 — SQLite `datetime('now','localtime')` 포맷(`YYYY-MM-DD HH:MM:SS`)이 lexicographic으로 정확하게 비교됨
- empty timestamp fallback: `importUpdatedAt == ""`이면 guard 스킵 — 레거시 MD import 하위 호환 보장

### Design
- `handleExport` → `ExportJSON`은 import-first 없는 순수 export. `autoExport()`가 DB 데이터를 역전시킬 위험 없음
- stale guard는 `SyncImport`(notify start), `SafeExportJSON`(workflow merge)의 import-first 경로를 방어

### Test
- StaleGuard: CLI `Update` → `UpdateFromImport` with old timestamp → title 유지 확인
- EmptyTimestamp: empty `importUpdatedAt` → import 정상 적용 확인
