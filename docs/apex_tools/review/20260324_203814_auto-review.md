## 개요

apex-agent 전체 Go 코드베이스(~20K LOC) 풀 리뷰. 5명의 서브에이전트 리뷰어(logic, test, design, systems, infra-security) 병렬 디스패치 후 cross-cutting 분석.

## 수정된 이슈 (9건)

### [HIGH] 1. NotifyDrop 트랜잭션 원자성 (3명 동시 발견: logic, design, systems)
- **파일**: `internal/modules/handoff/manager.go`
- **문제**: FIXING→OPEN 복귀와 finalizeBranch가 별도 트랜잭션으로 실행되어, 중간 crash 시 데이터 불일치 발생 가능
- **수정**: `finalizeBranch`에 `releaseFixing` 파라미터 추가, 단일 TX 안에서 OPEN 복귀 + history 이관 + 정리를 원자적으로 처리

### [MED] 2. NotifyStart getBacklogIDs 에러 무시 (logic)
- **파일**: `internal/modules/handoff/manager.go:90`
- **문제**: `oldIDs, _ := m.getBacklogIDs(tx, branch)` — DB 에러 시 이전 FIXING 백로그가 orphan 가능
- **수정**: 에러를 전파하여 트랜잭션 중단

### [MED] 3. queue.Acquire WAITING check+insert 비트랜잭션 (3명 동시 발견)
- **파일**: `internal/modules/queue/manager.go:130-139`
- **문제**: 동시 호출 시 중복 WAITING 엔트리 삽입 가능
- **수정**: `RunInTx` 안에서 check + insert 원자적 처리

### [MED] 4. backlog.Release 비트랜잭션 + 무조건 description append (logic, design)
- **파일**: `internal/modules/backlog/manage.go:394-428`
- **문제**: Get+Update+junctionCleaner가 비트랜잭션. FIXING 아닌 항목도 description 오염
- **수정**: `RunInTx`로 감싸고, FIXING 상태만 Release 허용 (non-FIXING은 에러 반환)

### [MED] 5. backlog.Fix 비트랜잭션 (design)
- **파일**: `internal/modules/backlog/manage.go:432-465`
- **문제**: Get+Update+INSERT가 비트랜잭션, TOCTOU 가능
- **수정**: `RunInTx`로 원자적 처리

### [MED] 6. backlog.Resolve 상태 미검증 (logic)
- **파일**: `internal/modules/backlog/manage.go:324-349`
- **문제**: 이미 RESOLVED인 항목을 다시 resolve 가능 → resolved_at/resolution 덮어쓰기
- **수정**: `AND status != 'RESOLVED'` 조건 추가, 중복 resolve 거부

### [MED] 7. validate-build 명령 치환 우회 (infra-security)
- **파일**: `internal/modules/hook/gate.go`
- **문제**: `echo $(ninja -C out)` — readOnly prefix + 명령 치환 결합 시 차단 우회
- **수정**: `hasCommandSubstitution()` 함수 추가, `$(`, 백틱, `<(` 포함 명령은 화이트리스트 제외

### [MED] 8. validate-build substring 매칭 부정확 (infra-security)
- **파일**: `internal/modules/hook/gate.go`
- **문제**: `strings.Contains(sub, "apex-agent")` — 환경변수 `VAR=apex-agent` 등으로 우회 가능. `hasGoToolCommand`도 `"mango build"` 같은 false positive
- **수정**: 화이트리스트 매칭을 토큰 basename 기반으로 변경. Go 도구 매칭을 `\bgo\s+(build|...)\b` 정규식으로 변경

### [MED] 9. IPC server Accept 에러 로깅 부재 (systems)
- **파일**: `internal/ipc/server.go:63-65`
- **문제**: Accept 실패 시 로그 없음 + tight loop 가능
- **수정**: `ml.Warn("accept error", ...)` 로깅 추가 + 50ms sleep으로 tight loop 방지
