## 보안 분석: fail-open 패턴

### 문서-코드 불일치 확인

CLAUDE.md에 "데몬 죽으면 모든 hook 차단"이라고 기재되어 있으나, 실제 코드는 데몬 미연결 시 `return nil`(허용)이다.

**해당 hook:**
- `validate-merge` (hook_cmd.go:62-72): 데몬 IPC 실패 → `return nil`
- `validate-handoff` (handoff_hook_cmd.go:49-50): 데몬 미연결 → 통과
- `handoff-probe` (handoff_hook_cmd.go:120-121): 데몬 미연결 → `return nil`

**판단**: 의도적 graceful degradation. fail-close로 전환하면 데몬 장애 시 모든 작업이 중단되는 더 큰 문제 발생. 현재 설계가 합리적이나 문서를 코드에 맞게 정정하는 것이 바람직하다. CLAUDE.md 내용을 "데몬 죽으면 데몬 의존 hook(merge lock, handoff)이 우회됨 — `daemon start`로 재시작 필수"로 수정 권고.
