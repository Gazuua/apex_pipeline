# BACKLOG-163: 백로그 CLI 강화 + JSON 포맷 통합 + Read 차단

## 목표

1. `backlog show <ID>` — 개별 항목 전체 필드 조회
2. `backlog list --verbose` — description 첫 줄 함께 출력
3. BACKLOG.md + BACKLOG_HISTORY.md → 단일 `docs/BACKLOG.json` 통합
4. Export/Import를 JSON 기반으로 전환
5. validate-backlog hook에 Read 차단 추가

## 배경

- DB(SQLite)가 source of truth, MD 파일은 git 백업
- MD 커스텀 포맷은 파싱 복잡도만 높이고 Read 차단 예정이라 가독성 불필요
- JSON은 Go encoding/json으로 제로 파싱 코드 + HTTP 대시보드(#146) 재활용 가능

## 설계

### 1. BACKLOG.json 스키마

```json
{
  "next_id": 163,
  "items": [
    {
      "id": 133,
      "title": "TransportContext의 ssl::context*...",
      "severity": "MAJOR",
      "timeframe": "IN_VIEW",
      "scope": "CORE, SHARED",
      "type": "DESIGN_DEBT",
      "status": "OPEN",
      "related": [130],
      "description": "TransportContext가..."
    },
    {
      "id": 153,
      "title": "...",
      "severity": "MINOR",
      "timeframe": "DEFERRED",
      "scope": "TOOLS",
      "type": "BUG",
      "status": "RESOLVED",
      "resolution": "FIXED",
      "resolved_at": "2026-03-23T21:30:00",
      "description": "..."
    }
  ]
}
```

- Pretty-print: `json.MarshalIndent(data, "", "  ")` — git diff 가독성 확보
- 정렬: OPEN 먼저 (NOW → IN_VIEW → DEFERRED, 각 내 severity 순), RESOLVED (resolve 날짜 역순)
- `related` 필드: comma-separated 문자열 (DB와 동일, 예: `"130,153"`). 정수 배열 전환은 DB 스키마 변경 동반하므로 향후 대시보드(#146) 구현 시 재평가

### 2. CLI 변경

#### `backlog show <ID>` (신규)

```
$ apex-agent backlog show 133

ID:          133
Title:       TransportContext의 ssl::context* — apex_core에 OpenSSL 직접 의존
Severity:    MAJOR
Timeframe:   IN_VIEW
Scope:       CORE, SHARED
Type:        DESIGN_DEBT
Status:      OPEN
Related:     #130
Description:
  TransportContext가 boost::asio::ssl::context*를 직접 보유하여...
```

RESOLVED 항목 추가 필드:
```
Resolution:  FIXED
Resolved:    2026-03-23 21:30:00
```

구현: CLI → IPC `backlog show {id}` → 데몬 manager.GetByID(id) → 응답

#### `backlog list --verbose` (개선)

기존 테이블에 description 첫 줄 추가:
```
ID     SEVERITY   SCOPE        TYPE        TITLE
  (description 첫 줄)
----------------------------------------------------------------------
133    MAJOR      CORE, SHARED DESIGN_DEBT TransportContext의 ssl::context*...
  TransportContext가 boost::asio::ssl::context*를 직접 보유하여...
```

구현: 기존 list 응답에 이미 Description 포함 — 출력 포맷만 변경

### 3. Export 변경

#### 기존
- `ExportBacklog()` → `docs/BACKLOG.md` (MD 포맷, OPEN만)
- `ExportHistory()` → `docs/BACKLOG_HISTORY.md` (MD 포맷, RESOLVED만)

#### 변경 후
- `Export()` → `docs/BACKLOG.json` (JSON, 전체 항목)
- `ExportBacklog()`, `ExportHistory()` 삭제
- `SafeExport()` 내부: import-first → 단일 Export()

### 4. Import 변경

#### 기존
- `ParseBacklogMD()` — BACKLOG.md 커스텀 MD 파서
- `ParseBacklogHistoryMD()` — BACKLOG_HISTORY.md 커스텀 MD 파서

#### 변경 후
- `ImportJSON()` — `json.Unmarshal` 한 줄
- 하위 호환: 마이그레이션 시 기존 MD 파일 존재하면 MD 파서로 import 후 JSON export → MD 삭제

### 5. 마이그레이션 전략

1. 기존 `docs/BACKLOG.md` + `docs/BACKLOG_HISTORY.md` import (기존 MD 파서 유지)
2. DB 기준으로 `docs/BACKLOG.json` export
3. 기존 MD 파일 2개 삭제 (`git rm`)
4. MD 파서 코드는 즉시 삭제하지 않음 — 마이그레이션 경로 유지 (다음 릴리즈에서 삭제)

### 6. Hook 변경

`gate.go` ValidateBacklog 확장:

| 도구 | 기존 | 변경 후 |
|------|------|---------|
| Edit/Write `docs/BACKLOG.md` | 차단 | 차단 (하위 호환) |
| Edit/Write `docs/BACKLOG_HISTORY.md` | 차단 | 차단 (하위 호환) |
| Edit/Write `docs/BACKLOG.json` | 없음 | 차단 |
| Read `docs/BACKLOG.md` | 허용 | 차단 + "backlog list/show 사용" 안내 |
| Read `docs/BACKLOG_HISTORY.md` | 허용 | 차단 + "backlog list/show 사용" 안내 |
| Read `docs/BACKLOG.json` | 없음 | 차단 + "backlog list/show 사용" 안내 |

hook_cmd.go에서 Read 이벤트(PreToolUse Read)도 ValidateBacklog 경유하도록 추가.

## 테스트

- `backlog show` 기본 동작 + 존재하지 않는 ID + RESOLVED 항목
- `backlog list --verbose` 출력 검증
- JSON export 라운드트립: DB → JSON → import → DB 비교
- 마이그레이션: MD 2파일 → JSON 1파일 변환 검증
- Read 차단 hook: BACKLOG.json, BACKLOG.md, BACKLOG_HISTORY.md 모두 차단 확인
