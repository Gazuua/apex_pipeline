# auto-review 플러그인 v1.1 설계

## 개요

auto-review 플러그인 3가지 개선:
1. reviewer-code / reviewer-test에 superpowers 코드리뷰 스킬 연동 + 2-tier 멀티에이전트 병렬 리뷰
2. Confidence threshold ≥ 80 → ≥ 40 하향
3. 재리뷰 시 Smart Skip 로직 (Clean + 무관 에이전트 스킵)

## 1. 2-Tier 멀티에이전트 구조 (reviewer-code, reviewer-test)

### 구조

```
오케스트레이터 → reviewer-code (1차 에이전트)
                  ├─ superpowers:code-reviewer (모듈 A)
                  ├─ superpowers:code-reviewer (모듈 B)
                  └─ superpowers:code-reviewer (모듈 C)
                  → 결과 취합 후 보고
```

### 동작 흐름

1. reviewer-code/test가 리뷰 범위 파악
2. 모듈/디렉토리 단위로 분할 (예: include/, src/, examples/)
3. 각 분할 단위마다 `superpowers:code-reviewer` 서브에이전트 병렬 디스패치
   - task 모드: git diff range + 해당 모듈 파일 전달
   - full 모드: 해당 모듈 전체 파일 목록 전달
4. 서브에이전트 결과 취합 → 중복 제거 → confidence ≥ 40 필터 → 최종 보고

### Fallback

superpowers 플러그인 미설치 시 → 현행 자율 리뷰 (기존 방식 유지).
에이전트 프롬프트에 "superpowers:code-reviewer 사용을 시도하라. 실패 시 자율 분할 리뷰로 진행하라" 명시.

## 2. Confidence Threshold 하향 (≥ 40)

### 변경 대상

5개 에이전트 전부: reviewer-code, reviewer-test, reviewer-docs, reviewer-structure, reviewer-general

### 스코어링 체계

```
0-25:  false positive 가능성 높음 → 제외
26-39: 불확실하지만 가능성 있음 → 제외
40-60: 유효하지만 저영향 → 보고 (Minor 추천)
61-80: 주의 필요 → 보고
81-100: 치명적 또는 명시적 규칙 위반 → 보고
```

### 2-Tier 에이전트에서의 적용

- superpowers 서브에이전트에게는 "가능한 많이 보고하라"
- 1차 에이전트(reviewer-code/test)가 취합 단계에서 confidence < 40 필터링

## 3. Smart Re-review (재리뷰 스킵 로직)

### 변경 대상

오케스트레이터 (`commands/auto-review.md`) Phase 2 → Phase 1 재진입 로직

### 스킵 판단 절차

1. 수정된 파일 목록 추출 (`git diff HEAD~1..HEAD --name-only`)
2. 파일 타입 매핑으로 "최소 필수 리뷰어" 결정:

| 수정 파일 타입 | 영향받는 리뷰어 |
|--------------|--------------|
| .cpp/.hpp (소스/헤더) | code, test, structure |
| test_*.cpp, test_helpers.hpp | test |
| CMakeLists.txt, vcpkg.json, build.*, CMakePresets | structure, general |
| *.md (문서) | docs |
| .gitignore, hooks, scripts | general |

3. 오케스트레이터가 수정 내용의 "의미적 영향" 추가 판단:
   - 코드 로직 변경 → docs 추가 (설계 정합성)
   - 의존성 변경 → general 추가
   - 구조 변경 → 전원 재리뷰
   - 판단 애매 → 포함 (보수적)

4. 스킵 조건 (AND):
   - 직전 라운드에서 Clean (0건)
   - 위 판단에서 "영향받는 리뷰어"에 미포함

### 예시 시나리오

| Round 1 결과 | 수정 내용 | Round 2 참여 |
|-------------|----------|-------------|
| docs만 이슈 | 문서 오탈자 수정 | docs만 / 나머지 스킵 |
| code + test 이슈 | server.cpp if문 추가 | 전원 (코드 로직 → 영향 넓음) |
| general만 이슈 | ci.yml 수정 | general + structure / 나머지 스킵 |

### 리뷰 보고서에 스킵 정보 포함

```markdown
## Round N 참여 현황
- reviewer-docs: 스킵 (Round N-1 Clean + 수정 무관)
- reviewer-code: 리뷰 수행
...
```
