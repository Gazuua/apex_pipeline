# cleanup 핫픽스 + 핸드오프 구조 강화 — 완료 기록

**브랜치**: `bugfix/cleanup-self-destruct`
**완료일**: 2026-03-23

## 배경

cleanup --execute 실행 시 현재 워크트리를 "머지 완료 브랜치"로 판단하고 디렉토리째 삭제하는 CRITICAL 버그 발견. 빈 커밋 없이 main에서 생성된 브랜치를 IsMergedToMain이 "머지 완료"로 판정 + 현재 워크트리/핸드오프 상태 확인 로직 부재가 원인.

## 구현 내용

### 1. TxStore 타입 분리
- `Store`에서 `tx *sql.Tx` 필드 제거, `TxStore` 별도 타입으로 분리
- `Querier` 인터페이스로 Store/TxStore 공통 사용 지원
- `MigrateFunc`, `BacklogOperator.SetStatusWith` 등 전 모듈 시그니처 일괄 변경

### 2. DB 스키마 마이그레이션 v5
- `branches` → `active_branches` 리네이밍 (활성 작업만)
- `branch_history` 신규 테이블 (머지/포기 이력)
- `git_branch` UNIQUE 제약 추가
- 기존 MERGE_NOTIFIED 레코드 → history(MERGED) 이관, 나머지 → history(DROPPED) 이관

### 3. MERGE_NOTIFIED 상태 폐기
- 활성 상태: STARTED, DESIGN_NOTIFIED, IMPLEMENTING (3개만)
- 머지/포기 시 active에서 삭제 → history로 이관 (MERGED/DROPPED)
- NextStatus에서 merge 전이 경로 제거, CanEditSource에서 MERGE_NOTIFIED 조건 제거

### 4. NotifyMerge / NotifyDrop
- `NotifyMerge`: active → history(MERGED) 이관 + FIXING 게이트
- `NotifyDrop`: active → history(DROPPED) 이관 + FIXING 게이트
- `checkFixingBacklogs` 공용 함수 (merge/drop/ValidateMergeGate 통합)
- `finalizeBranch` 헬퍼로 merge/drop 공통 TX 로직 추출
- `ListActive` IPC 라우트 (cleanup 연동용)
- `txs.Exec` 에러 무시 버그 수정

### 5. CLI 변경
- `notify merge`: notify-merge IPC + Phase 2 git checkout main
- `notify drop`: 신규 커맨드 + Phase 2 git checkout main
- `notify start`: `--branch-name` 필수 + Phase 0 git 존재 확인 + Phase 2 git 생성
- `--scopes` 필수로 격상, `doNotifyStart` 공용 함수 추출

### 6. cleanup 연동
- `Run()` 시그니처에 `activeBranches map[string]bool` 추가
- 3개 Phase 모두 activeBranches 체크 → 활성 핸드오프 브랜치 스킵
- `isSubPath` CWD 보호 (defense-in-depth)
- `cleanup_cmd.go`에서 list-active IPC로 활성 브랜치 조회

### 7. validate-build 확장
- `git checkout -b/-B`, `git switch -c/--create`, `git branch <name>`, `git worktree add -b` 차단
- 조회/삭제 명령은 허용 유지

### 8. 백로그 등록
