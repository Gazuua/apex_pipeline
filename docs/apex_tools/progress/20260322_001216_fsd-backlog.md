# BACKLOG-123: FSD Backlog 소탕 자동화 시스템

**브랜치**: `feature/backlog-automation`
**날짜**: 2026-03-22

## 작업 요약

`/fsd-backlog` 슬래시 커맨드를 구현하여 백로그 소탕을 완전 자율 수행할 수 있게 했다.

## 산출물

| 파일 | 동작 | 설명 |
|------|------|------|
| `apex_tools/claude-plugin/commands/fsd-backlog.md` | 생성 | 슬래시 커맨드 스킬 프롬프트 |
| `apex_tools/claude-plugin/.claude-plugin/plugin.json` | 수정 | 설명·버전 갱신 (3.0.0→3.1.0) |

## 설계 결정

- **스킬 기반 오케스트레이터**: 기존 인프라(branch-handoff, queue-lock, auto-review) 100% 재활용. 새 스크립트 없이 프롬프트 파일 1개로 구현
- **핸드오프 가이드 내장**: 스킬 프롬프트에 정확한 명령어 순서를 포함하여 에이전트가 `init` vs `notify start` 등으로 헤매는 문제 방지
- **CLAUDE.md 비중복**: 기존 워크플로우 규칙을 재기술하지 않고, FSD 고유 행동(스캔/선별/드롭 판단)만 정의
- **plugin.json 이름 유지**: `apex-auto-review` → 변경 시 setup-claude-plugin.sh 등 연쇄 수정 필요하여 현행 유지
