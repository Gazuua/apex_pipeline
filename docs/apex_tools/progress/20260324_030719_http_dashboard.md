# HTTP 대시보드 + 워크플로우 컨트롤 완료

> BACKLOG-154, BACKLOG-159 | PR #130 | 브랜치: feature/http-dashboard-workflow

## 요약

apex-agent 데몬에 HTTP 서버를 내장하여 브라우저 기반 실시간 시스템 상태 대시보드를 구현.
추가로 빌드 lock PID 자식 프로세스 이관 버그 수정 및 Queue finished_at 컬럼 추가.

## 주요 변경

### HTTP 대시보드 (BACKLOG-154, BACKLOG-159)

- **httpd 패키지 신규** — `internal/httpd/` (server, routes, handlers, queries, render, templates, static)
- **데몬 내장** — IPC 서버와 병렬로 HTTP 서버 기동 (`localhost:7600`)
- **Go 템플릿 + HTMX 2.0.4** — 서버 사이드 렌더링, JS 프레임워크 없음
- **다크 테마** — 쇼케이스 페이지(`agent_ecosystem.html`) 디자인 토큰 계승
- **4개 페이지**: Dashboard, Backlog, Handoff, Queue
- **실시간 갱신** — 1초 폴링 (Fast 0.5s / Normal 1s / Slow 2s 조절 가능, localStorage 저장)
- **JSON API** — `/api/backlog`, `/api/handoff`, `/api/queue`
- **config** — `[http]` 섹션 (enabled, addr)

### Backlog 페이지

- 다중 필터 (체크박스 — Status, Severity, Timeframe)
- 기본 정렬 (FIXING → NOW → CRITICAL → ID)
- 컬럼 헤더 클릭 정렬 (방향 화살표 ↑↓⇅ 표시)
- 행 클릭 시 상세 펼침 (아코디언 — 하나만)
- RESOLVED 항목 timeframe "—" 표시

### Handoff 페이지

- 상태 머신 시각화 (started → design → implementing → merge)
- 워크스페이스 표시
- 백로그 번호 클릭 시 HTMX로 상세 인라인 펼침
- 브랜치 히스토리 (최신 먼저)

### Queue 페이지

- 채널별 잠금 상태 (Build/Merge)
- ACTIVE/WAITING: 실시간 경과 시간 (1초 갱신)
- DONE: 점유 시간 표시 (finished_at - created_at)
- 상태별 색상 (ACTIVE 빨강, WAITING 노랑, DONE 파랑, Free 초록)

### 빌드 lock PID 수정

- **문제**: `apex-agent queue build`의 parent PID로 lock 기록 → parent 죽으면 lock 해제 → 빌드는 고아로 계속
- **수정**: `Run()` → `Start()` + child PID로 DB 갱신 + `Wait()`
- `Manager.UpdatePID()` 메서드 + `update-pid` IPC 핸들러 추가

### Queue finished_at

- migration v3: `finished_at` TEXT 컬럼 추가
- `Release()` 시 `finished_at = datetime('now','localtime')` 기록
- Duration 계산: `julianday(finished_at) - julianday(created_at)`
