**내용**: Redis Pub/Sub 메시지 수신 시 `reply->element[1]->str` 및 `reply->element[2]->str`의 null 체크가 누락되어 있었음. Redis 프로토콜상 message 타입에서 이 필드들은 항상 문자열이어야 하지만, 네트워크 손상이나 비정상 Redis 응답 시 null이 될 수 있음. null인 `char*`로 `std::string_view`나 `std::span`을 구성하면 C++20 표준에서 undefined behavior.
