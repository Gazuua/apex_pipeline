- **수정**: 실제 vcpkg.json 기준으로 의존성 목록 동기화.

#### C-2. Redis cancel_all_pending UAF — concurrency

- **파일**: `apex_shared/lib/adapters/redis/src/redis_multiplexer.cpp:322-338`
- **리뷰어**: concurrency
- **문제**: `cancel_all_pending()` 에서 pending 큐 순회 중 promise 완료 콜백이 큐를 재진입할 수 있어 Use-After-Free 발생 가능. 코루틴 재개 시점에 이미 해제된 메모리 접근.
- **수정**: 큐를 로컬로 swap한 뒤 순회하도록 변경하여 재진입 안전성 확보.

#### C-3. PubSub 바이트 오더 LE→BE 불일치 — logic

- **파일**: `apex_services/chat-svc/src/chat_service.cpp:137-154`
- **리뷰어**: logic
- **문제**: PubSub 메시지 직렬화 시 Little Endian으로 인코딩하나, 수신 측에서 Big Endian으로 디코딩. 멀티바이트 필드(채널 ID, 타임스탬프 등)가 깨짐.
- **수정**: 인코딩/디코딩 양측을 Big Endian으로 통일.

#### C-4. JWT claim 타입 string→int + email→sub 불일치 — logic

- **파일**: `apex_services/auth-svc/src/jwt_manager.cpp:59-60`
- **리뷰어**: logic
- **문제**: JWT `exp`/`iat` claim을 문자열로 설정하고 있었으나 RFC 7519는 NumericDate (정수) 요구. 또한 사용자 식별자를 `email` claim에 넣고 있었으나 Gateway는 `sub` claim을 참조하여 인증 실패.
- **수정**: claim 타입을 정수로 변경, `email` → `sub` claim 사용으로 수정.

#### C-5. Gateway JWT issuer 하드코딩 불일치 — cross-cutting

- **파일**: `apex_services/gateway/include/.../gateway_config.hpp`, `apex_services/gateway/src/jwt_verifier.cpp`, `apex_services/gateway/src/gateway_config_parser.cpp`, `apex_infra/docker/gateway.toml`
- **리뷰어**: cross-cutting
- **문제**: Gateway의 JWT issuer가 `"apex"`로 하드코딩되어 있으나, auth-svc는 `"apex-auth"`를 issuer로 발행. issuer 검증 실패로 모든 인증 토큰이 거부됨.
- **수정**: Gateway config에서 issuer를 설정 파일(`gateway.toml`)로부터 읽도록 변경, 기본값을 `"apex-auth"`로 일치화.
