### 에스컬레이션 수정 — 메인 오케스트레이터 (5건)

#### Critical (3건 — Cross-Cutting 서비스 간 계약 위반)

10. **JWT 블랙리스트 키/값 불일치** (서비스 간 계약 위반)
    - Auth 서비스: `auth:blacklist:{SHA256(token)}` 키로 블랙리스트 저장
    - Gateway 서비스: `jwt:blacklist:{jti}` 키로 블랙리스트 조회
    - 키 형식이 완전히 달라 블랙리스트가 작동하지 않음 — **로그아웃한 토큰이 여전히 유효**
    - jti 기반으로 통일: 양쪽 모두 `jwt:blacklist:{jti}` 키 사용으로 수정

11. **Whisper Redis 세션 키 불일치** (서비스 간 계약 위반)
    - Auth 서비스: `auth:session:{uid}` 키로 세션 저장
    - Chat 서비스: `session:user:{uid}` 키로 세션 조회
    - 키 불일치로 Chat 서비스가 세션을 찾지 못함 — **귓속말 기능 불가**
    - `SessionStore`에 `session:user:{uid}` 매핑 추가하여 호환성 확보

12. **RefreshToken msg_id=10 라우팅 오류** (서비스 간 계약 위반)
    - RefreshToken에 `msg_id=10` 할당 — 시스템 예약 범위(0-999)에 해당
    - Gateway가 시스템 메시지로 분류하여 Auth 서비스로 라우팅하지 않음
    - auth 범위 `msg_id=1004`로 변경, 14개 관련 파일 갱신

#### Important (2건)

13. **auth_exempt/sensitive_msg_ids 모순 해소**
    - `msg_id=10`(RefreshToken)이 `auth_exempt`이면서 동시에 `sensitive_msg_ids`에 포함
    - 인증 면제 메시지가 민감 메시지로 분류되는 논리적 모순
    - msg_id 변경(→1004)에 맞춰 `sensitive_msg_ids` 비움으로 모순 해소

14. **std::stoull 예외 미처리 → std::from_chars 전환**
    - `auth_service` 2곳 + `chat_service` 7곳에서 `std::stoull` 사용 — 잘못된 입력 시 `std::invalid_argument` / `std::out_of_range` 예외 발생
    - 예외 없는 `std::from_chars`로 전환, `chat_service`에는 `safe_parse_u64` 헬퍼 도입
