## 발견 이슈 및 수정

### [CRITICAL] namespace-scope static ScopedLogger — init_logging() 전 생성으로 영구 no-op

**문제**: 19개 .cpp 파일에서 anonymous namespace에 `ScopedLogger s_logger{...}` 를
namespace-scope static으로 선언. C++ 동적 초기화 순서에 의해 `main()` 진입 전에
`spdlog::get()` 호출 → 아직 로거 미등록 → `logger_ = nullptr` 고정 → 모든 로그 영구 no-op.

**영향**: Session close/write_pump, FrameCodec 디코딩 오류, 어댑터 에러 등 운영 핵심 로그 미출력.

**수정**: function-scope static으로 전환 (C++11 Magic Statics — 첫 호출 시 초기화):
```cpp
// Before:
ScopedLogger s_logger{"Session", ScopedLogger::NO_CORE};

// After:
const ScopedLogger& s_logger()
{
    static const ScopedLogger instance{"Session", ScopedLogger::NO_CORE};
    return instance;
}
```

**수정 파일 (19개)**:
- apex_core: session, cross_core_dispatcher, frame_codec, config, periodic_task_scheduler,
  error_sender, arena_allocator, slab_allocator, ring_buffer, wire_header
- apex_shared: adapter_error, cancellation_token, hiredis_asio_adapter, consumer_payload_pool
- apex_services: jwt_verifier, route_table, gateway_config_parser, chat_service, jwt_manager

### [MEDIUM] GatewayPipeline — [sess=N] 구조화 태그 미사용

**문제**: `gateway_pipeline.hpp`의 `process()`와 `gateway_service.cpp`에서 `session->id()`를

## 이슈가 아닌 것으로 확인된 항목

- **with_trace() copy semantics**: const 메서드 + copy → 원본 불변성 보장, shared_ptr refcount만 증가
- **동시성 안전성**: fmt::memory_buffer 스택 로컬, spdlog logger thread-safe, 멤버 불변
- **null logger 안전성**: should_log()이 nullptr 체크 수행, default 생성 시 모든 호출 no-op
- **예외 안전성**: consteval log_loc이 compile-time format 검증 → 런타임 format_error 불가
- **spdlog 헤더 의존성**: should_log() 인라인 fast-path에 spdlog 타입 필요, pimpl 전환 불합리
- **Gateway 독립성 원칙**: ScopedLogger 도입이 Gateway의 도메인 무관 원칙 위반 안 함
- **ServiceBase::logger_ protected**: CRTP 구조에서 사실상 Derived만 접근, pit of success 유도
