## 작업 결과

### BACKLOG-116: apex_core → apex_shared 역방향 의존 제거

`apex_core` 라이브러리가 `apex_shared` 프로토콜 헤더를 PUBLIC include로 노출하던 레이어 위반을 제거했다.

**변경 내용**:
- `apex_core/CMakeLists.txt`에서 apex_shared 프로토콜 PUBLIC include path 2줄 제거 (tcp, kafka)
- 포워딩 헤더 `apex_core/include/apex/core/tcp_binary_protocol.hpp`에 `@deprecated` 주석 추가 (이후 최종 삭제됨)
- 소비자 10개 파일: `<apex/core/tcp_binary_protocol.hpp>` → `<apex/shared/protocols/tcp/tcp_binary_protocol.hpp>` 직접 include로 전환
- CMake 타겟 10개: `apex::shared::protocols::tcp` 명시적 PRIVATE 링크 추가

**효과**: apex_core는 이제 apex_shared 프로토콜에 대한 PUBLIC 의존성이 없다. 소비자가 필요한 프로토콜을 직접 링크하는 올바른 의존 방향으로 전환.

### BACKLOG-117: GatewayService services[0] 하드코딩 + dynamic_cast 제거

`gateway_service.cpp:257`에서 `dynamic_cast<GatewayService*>(state.services[0].get())` 패턴을 `state.registry.find<GatewayService>()`로 교체했다.

**변경 내용**:
- `gateway_service.cpp` 1곳: 위치 기반 조회(`services[0]`) → 타입 기반 조회(`registry.find<T>()`)
- 에러 메시지를 등록 순서 가정 없는 내용으로 갱신

**효과**: 서비스 등록 순서와 무관하게 동작하며, 프레임워크 가이드 D1(타입 기반 서비스 접근) 원칙을 준수.
