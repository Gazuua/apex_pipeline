# 보안 소탕 — Kafka manual commit + Gateway 인증 + NetworkPolicy

- **PR**: #207
- **백로그**: BACKLOG-247, 248, 249, 253
- **브랜치**: feature/security-sweep-20260326_215439

## 해결 항목

### BACKLOG-247: Kafka Consumer manual commit
- `enable.auto.commit=false`로 전환
- 처리 성공 또는 DLQ 라우팅 성공 후에만 오프셋 커밋
- DLQ produce 3회 재시도, 전부 실패 시 미커밋 → 재전달 (at-least-once)

### BACKLOG-248: Gateway auth exempt endpoint rate limit
- auth exempt msg_id(Login 등)에 per-endpoint rate limit 적용
- IP 주소를 FNV-1a 해시로 변환하여 synthetic user_id로 사용
- 기존 `[rate_limit.endpoint.overrides]` 설정 그대로 활용

### BACKLOG-249: Gateway 시스템 메시지 인증 필수화
- SUBSCRIBE/UNSUBSCRIBE에 인증 체크 추가 (미인증 시 거부)
- 시스템 메시지에도 per-IP rate limit 적용
- AUTHENTICATE_SESSION은 chicken-and-egg 문제로 정당하게 면제

### BACKLOG-253: K8s NetworkPolicy
- 4개 서비스(gateway, auth-svc, chat-svc, log-svc)에 NetworkPolicy 템플릿 추가
- Deny-All ingress/egress + 서비스별 최소 allow 규칙
- `networkPolicy.enabled` 토글 (로컬 false, prod true)
