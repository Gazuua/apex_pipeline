# v0.6.1 퀄리티 강화 — 완료

- **브랜치**: `feature/v0.6.1-quality-hardening`
- **PR**: #137
- **백로그**: BACKLOG-185, BACKLOG-186, BACKLOG-173

## 완료 항목

### BACKLOG-186: MetricsHttpServer HTTP read timeout
- `tcp::socket` → `beast::tcp_stream` 전환
- `stream.expires_after(5s)` 읽기/쓰기 양쪽 적용
- slowloris 공격 및 hang 클라이언트 방어

### BACKLOG-185: MetricsHttpServer in-flight session drain
- `MetricsSessionTracker` 도입 (세션 추적)
- `SessionGuard` RAII로 등록/해제 보장
- `stop()`에서 active stream cancel (셧다운 중 serialize 경합 방지)

### BACKLOG-173: Design 현대화 (4건)
1. **SFINAE→requires**: `server.hpp` `add_adapter` 오버로드 — `enable_if_t` → `requires`절
2. **RouteTable 통일**: `RouteTable::Entry` 제거 → `RouteEntry` 단일 타입, `build()`에서 벡터 직접 이동
3. **ErrorCode 순서**: enum + switch문 모두 값 순서(1~18)로 재배치
4. **CRTP 통일**: `wire_services`/`register_metrics` → `do_wire_services`/`do_register_metrics` 명시적 CRTP dispatch (KafkaAdapter, RedisAdapter, PgAdapter)
