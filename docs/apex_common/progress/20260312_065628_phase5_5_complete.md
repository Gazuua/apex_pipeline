## 구현 요약

### Tier 0: 벤치마크 인프라
- Google Benchmark 통합, bench_main + SystemProfile
- Micro 벤치마크 7개: mpsc_queue, ring_buffer, frame_codec, dispatcher, timing_wheel, slab_pool, session_lifecycle
- Integration 벤치마크 4개: cross_core_latency, frame_pipeline, session_throughput, cross_core_message_passing

### Tier 0.5: 에러 기반 정비
- DispatchError/QueueError → ErrorCode 통일
- async_send Result 반환 전환
- ErrorSender (에러 응답 프레임 빌더)

### Tier 1: 핫패스 + Cross-Core Message Passing
- drain/tick 분리 (boost::asio::post + atomic coalescing + batch limit)
- Cross-Core Message Passing 아키텍처 전환 (closure shipping → op 기반 handler dispatch)
- CrossCoreDispatcher + CrossCoreOp + SharedPayload
- MessageDispatcher std::array<65536> → boost::unordered_flat_map
- payload zero-copy dispatch (consume 순서 변경)

### Tier 1.5: E2E 부하 테스터
- echo_loadtest (멀티스레드 클라이언트, 워밍업/측정 구간 분리, p50/p90/p99 레이턴시)
- JSON before/after 비교 도구 (compare_benchmarks.py)

### Tier 2: 메모리 아키텍처
- SessionPtr shared_ptr → intrusive_ptr (non-atomic refcount)
- Session SlabPool 할당 (heap fallback 포함)
- sessions_/timer_to_session_ → boost::unordered_flat_map

### Tier 2.5: 시각화 + 보고서
- 벤치마크 시각화/비교/보고서 통합 도구 (generate_benchmark_report.py)

### Tier 3: 구조 + I/O
- Server → Server + ConnectionHandler 분리
- accept_io_ → control_io_ 리네이밍
- SO_REUSEPORT per-core acceptor (Linux)
- io_uring CMake 옵션 (APEX_USE_IO_URING)
- SlabPool auto-grow (SlabPoolConfig + grow_count/peak_allocated 메트릭)
