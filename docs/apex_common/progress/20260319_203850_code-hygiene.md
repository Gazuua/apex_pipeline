### Phase 별 성과

**Phase 1 — 포맷팅 (BACKLOG-58)**
- `.clang-format` 설정: Allman brace (람다 K&R 예외), 120자, 4칸 인덴트, LLVM 기반
- `.clang-format-ignore`: 생성 파일 (`*_generated.h`) 제외
- 전체 274 소스 파일 일괄 포맷팅
- `.git-blame-ignore-revs`: 포맷팅 커밋 blame 제외 등록
- CI `format-check` job 추가: `clang-format --dry-run --Werror`
- clang-format 버전 21.1.8 고정 (로컬/CI 불일치 방지)

**Phase 2 — 경고 소탕 (BACKLOG-54)**
- `cmake/ApexWarnings.cmake`: `apex_set_warnings()` 함수 정의
  - MSVC: `/W4 /WX` + 외부 라이브러리 `/external:I /external:W0`
  - GCC/Clang: `-Wall -Wextra -Wpedantic -Werror`
- 전 타겟(core, shared, services, tests, benchmarks)에 `apex_set_warnings()` 적용
- MSVC 경고 수정: `/W4` 전수 소탕 (unused parameter, missing-field-initializers 등)
- GCC 경고 수정: `missing-field-initializers` (designated initializer `{}` → 명시적 기본값), `unused-parameter`, `redundant-move`
- TSAN `atomic_thread_fence` false positive 억제
- PerCoreState 불완전 타입 해소

**부수 수정 — FileWatcher flaky 테스트 (#62)**
- `test_file_watcher.cpp` DetectsChange: NTFS 타임스탬프 캐싱으로 인한 간헐 실패 수정
- 초기 파일 `last_write_time`을 2초 과거로 고정하여 재작성 후 타임스탬프 차이 보장
