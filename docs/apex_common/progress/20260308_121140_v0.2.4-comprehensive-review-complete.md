## 3. Critical 수정 내역 (7건)

### C-1. IOCP 소켓 바인딩 수정 (프레임워크)
- **문제**: accept_io_에서 생성된 소켓이 core별 io_context로 이관되지 않아 멀티코어 분산이 사실상 무효화
- **수정**: TcpAcceptor에 `ContextProvider` 콜백 추가, `async_accept(target_ctx, token)` 사용으로 소켓을 처음부터 target core의 IOCP에 바인딩
- **파일**: `tcp_acceptor.hpp/cpp`, `server.cpp`

### C-2. process_frames payload 안전성 (프레임워크)
- **문제**: `co_await dispatcher.dispatch()` 중 RingBuffer 내부를 가리키는 payload span이 무효화될 수 있음
- **수정**: `consume_frame()`을 dispatch 전에 호출, payload를 로컬 vector로 복사
- **파일**: `server.cpp`

### C-3. FrameError 미처리 → 무한 파싱 루프 (네트워크)
- **문제**: `try_decode`의 `BodyTooLarge`/`HeaderParseError` 시 세션 close 안 하고 버퍼에 악성 데이터가 남음
- **수정**: 에러 종류별 분기 — `InsufficientData`면 break, 그 외면 `session->close()` + `co_return`
- **파일**: `server.cpp`

### C-4. RingBuffer linearize() span 수명 미문서화 (기반)
- **문제**: contiguous→buffer 포인터, wrap→linear_buf_ 포인터 — consume/다음 linearize 시 댕글링
- **수정**: `@warning` 주석 추가 + `write()` 편의 메서드 추가로 수동 split-write 제거
- **파일**: `ring_buffer.hpp/cpp`

### C-5. 버전 불일치 0.2.3 vs 0.2.4 (빌드)
- **문제**: 루트 CMakeLists.txt, vcpkg.json이 0.2.3, apex_core는 0.2.4
- **수정**: 루트 `VERSION 0.2.4`, `"version-semver": "0.2.4"` 동기화
- **파일**: `CMakeLists.txt`, `vcpkg.json`

### C-6. Apex_Pipeline.md 프로젝트 구조 옛 경로 (문서)
- **문제**: `apex_core/docs/`로 기술 — 실제는 `docs/apex_core/`
- **수정**: 구조도 현행화 + 4개 문서의 깨진 경로 참조 전량 수정
- **파일**: `docs/Apex_Pipeline.md`, `docs/apex_core/plans/*.md`, `docs/apex_core/progress/*.md`

### C-7. 미사용 vcpkg 의존성 (빌드)
- **문제**: spdlog, tomlplusplus, benchmark — find_package도 include도 없이 선언만 존재
- **수정**: 3개 의존성 제거 (실제 사용 시 재추가)
- **파일**: `vcpkg.json` (루트+apex_core)
