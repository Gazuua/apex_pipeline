# v0.4.3 PostgreSQL 어댑터 — Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** libpq 비동기 모드를 Boost.Asio에 직접 통합하여, 코루틴 기반 비동기 PostgreSQL 쿼리를 지원하는 PgAdapter/PgPool을 구현한다. libpq 소켓 fd를 Asio 이벤트 루프에 등록하여 단일 코루틴 모델 + 락프리 구조를 유지하는 것이 포트폴리오 핵심 기술 포인트.

**Architecture:** PgAdapter는 AdapterBase CRTP를 상속하는 전역 1개 인스턴스. 내부에 코어별 PgPool(ConnectionPool CRTP 상속)을 소유하며, `CoreEngine::current_core_id()`로 현재 코어의 풀을 자동 라우팅한다. PgPool은 libpq의 `PQconnectStart()` → `PQsocket()` → Asio fd 등록 → `PQconnectPoll()` 비동기 커넥션과, `PQsendQuery()` → Asio readable → `PQgetResult()` 비동기 쿼리 파이프라인을 코루틴(`awaitable<>`)으로 래핑한다.

**Tech Stack:** C++23, Boost.Asio, libpq (vcpkg), Google Test, CMake

**Spec:** `docs/apex_common/plans/20260313_013858_v0.4_external_adapters_design.md`

**Depends on:** Plan 1 (v0.4.0.1 공통 추상화 — AdapterBase, ConnectionPool, AdapterError, Server add_adapter API)

## Task 2: libpq Asio 어댑터 (PgConnection)

**Goal:** libpq의 비동기 모드를 Boost.Asio 이벤트 루프에 통합하는 `PgConnection` 클래스를 구현한다. 이것이 v0.4.3의 기술적 핵심이며 포트폴리오 하이라이트.

**Files:**
- Create: `apex_shared/lib/adapters/pg/include/apex/shared/adapters/pg/pg_connection.hpp`
- Create: `apex_shared/lib/adapters/pg/include/apex/shared/adapters/pg/pg_result.hpp`
- Implement: `apex_shared/lib/adapters/pg/src/pg_connection.cpp`

### 핵심 설계

libpq 비동기 모드 + Asio fd 등록 파이프라인:

```
[비동기 커넥션]
PQconnectStart(conninfo)
  → PQsocket(conn) → fd 획득
  → boost::asio::ip::tcp::socket::assign(tcp::v4(), fd)  (Windows IOCP 등록)
  → PQsetnonblocking(conn, 1)
  → loop:
      PQconnectPoll(conn)
        → PGRES_POLLING_READING  → co_await socket.async_wait(wait_read)
        → PGRES_POLLING_WRITING  → co_await socket.async_wait(wait_write)
        → PGRES_POLLING_OK       → 완료
        → PGRES_POLLING_FAILED   → 에러

[비동기 쿼리]
PQsendQuery(conn, sql)  또는  PQsendQueryParams(conn, sql, ...)
  → loop:
      co_await socket.async_wait(wait_read)
      → PQconsumeInput(conn)
      → while PQisBusy(conn) == false:
          PQgetResult(conn) → PgResult 수집
      → PQgetResult(conn) == nullptr → 쿼리 완료
```

```cpp
// apex_shared/lib/adapters/pg/include/apex/shared/adapters/pg/pg_result.hpp
#pragma once

#include <libpq-fe.h>

#include <cstdint>
#include <memory>
#include <string_view>

namespace apex::shared::adapters::pg {

/// PGresult의 RAII 래퍼. PQclear()를 자동 호출.
class PgResult {
public:
    PgResult() = default;
    explicit PgResult(PGresult* result) : result_(result, &PQclear) {}

    /// 쿼리 성공 여부 (PGRES_COMMAND_OK 또는 PGRES_TUPLES_OK)
    [[nodiscard]] bool ok() const noexcept;

    /// PG 에러 메시지
    [[nodiscard]] std::string_view error_message() const noexcept;

    /// 결과 상태
    [[nodiscard]] ExecStatusType status() const noexcept;

    /// 행/열 수
    [[nodiscard]] int row_count() const noexcept;
    [[nodiscard]] int column_count() const noexcept;

    /// 셀 값 접근 (text 형식)
    [[nodiscard]] std::string_view value(int row, int col) const noexcept;
    [[nodiscard]] bool is_null(int row, int col) const noexcept;

    /// 컬럼명
    [[nodiscard]] std::string_view column_name(int col) const noexcept;

    /// 영향받은 행 수 (INSERT/UPDATE/DELETE)
    [[nodiscard]] int affected_rows() const noexcept;

    /// 유효한 결과가 있는지
    [[nodiscard]] explicit operator bool() const noexcept { return result_ != nullptr; }

    /// 원본 PGresult 접근 (escape hatch)
    [[nodiscard]] PGresult* get() const noexcept { return result_.get(); }

private:
    std::unique_ptr<PGresult, decltype(&PQclear)> result_{nullptr, &PQclear};
};

} // namespace apex::shared::adapters::pg
```

- [ ] **Step 2: PgConnection 헤더 작성**

```cpp
// apex_shared/lib/adapters/pg/include/apex/shared/adapters/pg/pg_connection.hpp
#pragma once

#include <apex/shared/adapters/pg/pg_result.hpp>
#include <apex/shared/adapters/pg/pg_config.hpp>
#include <apex/shared/adapters/adapter_error.hpp>
#include <apex/core/result.hpp>

#include <libpq-fe.h>

#include <boost/asio/awaitable.hpp>
#include <boost/asio/ip/tcp.hpp>
#include <boost/asio/io_context.hpp>

#include <memory>
#include <span>
#include <string>
#include <string_view>
#include <vector>

namespace apex::shared::adapters::pg {

/// libpq 비동기 커넥션을 Asio에 통합한 래퍼.
///
/// 핵심: PQsocket() fd를 boost::asio::ip::tcp::socket에 assign하여
/// IOCP(Windows)/epoll(Linux)에 등록. 모든 I/O를 Asio 이벤트 루프에서 처리.
///
/// 사용 패턴:
///   PgConnection conn(io_ctx);
///   co_await conn.connect_async(conninfo);
///   auto result = co_await conn.query_async("SELECT ...");
///
/// 스레드 안전성: NOT thread-safe. 코어별 PgPool에서 단일 코어 스레드만 접근.
class PgConnection {
public:
    explicit PgConnection(boost::asio::io_context& io_ctx);
    ~PgConnection();

    // Non-copyable, movable
    PgConnection(const PgConnection&) = delete;
    PgConnection& operator=(const PgConnection&) = delete;
    PgConnection(PgConnection&& other) noexcept;
    PgConnection& operator=(PgConnection&& other) noexcept;

    /// 비동기 커넥션 수립.
    /// PQconnectStart → PQsocket → Asio assign → PQconnectPoll 루프.
    /// @param conninfo libpq 커넥션 문자열 (예: "host=localhost port=6432 dbname=apex")
    [[nodiscard]] boost::asio::awaitable<apex::core::Result<void>>
    connect_async(std::string_view conninfo);

    /// 비동기 쿼리 실행 (파라미터 없음).
    /// PQsendQuery → Asio readable → PQconsumeInput → PQgetResult.
    [[nodiscard]] boost::asio::awaitable<apex::core::Result<PgResult>>
    query_async(std::string_view sql);

    /// 비동기 파라미터 쿼리 실행.
    /// PQsendQueryParams → Asio readable → PQconsumeInput → PQgetResult.
    /// @param sql SQL with $1, $2, ... placeholders
    /// @param params 파라미터 값 (text 형식)
    [[nodiscard]] boost::asio::awaitable<apex::core::Result<PgResult>>
    query_params_async(std::string_view sql, std::span<const std::string> params);

    /// 비동기 명령 실행 (결과 행 없는 INSERT/UPDATE/DELETE).
    /// 내부적으로 query_async()를 호출하되, 결과에서 affected_rows만 반환.
    [[nodiscard]] boost::asio::awaitable<apex::core::Result<int>>
    execute_async(std::string_view sql);

    /// 비동기 파라미터 명령 실행.
    [[nodiscard]] boost::asio::awaitable<apex::core::Result<int>>
    execute_params_async(std::string_view sql, std::span<const std::string> params);

    /// 커넥션 유효성 검증 (PQstatus).
    [[nodiscard]] bool is_valid() const noexcept;

    /// 커넥션 닫기. fd 해제 + PQfinish.
    void close() noexcept;

    /// 커넥션 상태
    [[nodiscard]] bool is_connected() const noexcept;

private:
    /// PQconnectPoll 루프 (connect_async 내부)
    [[nodiscard]] boost::asio::awaitable<apex::core::Result<void>>
    poll_connect();

    /// 쿼리 결과 수신 루프 (query_async/execute_async 내부)
    [[nodiscard]] boost::asio::awaitable<apex::core::Result<PgResult>>
    collect_result();

    /// Asio 소켓에서 libpq fd 해제 (소켓은 libpq가 관리)
    void release_socket() noexcept;

    /// AdapterError 생성 헬퍼
    [[nodiscard]] static apex::core::ErrorCode make_error();

    boost::asio::io_context& io_ctx_;
    PGconn* conn_ = nullptr;
    /// libpq fd를 Asio에 등록하기 위한 소켓.
    /// Windows: tcp::socket::assign(tcp::v4(), PQsocket(conn)) → IOCP 등록
    /// Linux:   tcp::socket::assign(tcp::v4(), PQsocket(conn)) → epoll 등록
    std::unique_ptr<boost::asio::ip::tcp::socket> socket_;
    bool connected_ = false;
};

} // namespace apex::shared::adapters::pg
```

- [ ] **Step 3: pg_result.cpp 구현**

`pg_connection.cpp` 내에 PgResult 구현을 포함하거나 별도 파일로 분리:

```cpp
// PgResult 멤버 구현
bool PgResult::ok() const noexcept {
    if (!result_) return false;
    auto s = PQresultStatus(result_.get());
    return s == PGRES_COMMAND_OK || s == PGRES_TUPLES_OK;
}

ExecStatusType PgResult::status() const noexcept {
    return result_ ? PQresultStatus(result_.get()) : PGRES_FATAL_ERROR;
}

std::string_view PgResult::error_message() const noexcept {
    if (!result_) return "no result";
    const char* msg = PQresultErrorMessage(result_.get());
    return msg ? msg : "";
}

int PgResult::row_count() const noexcept {
    return result_ ? PQntuples(result_.get()) : 0;
}

int PgResult::column_count() const noexcept {
    return result_ ? PQnfields(result_.get()) : 0;
}

std::string_view PgResult::value(int row, int col) const noexcept {
    if (!result_ || PQgetisnull(result_.get(), row, col)) return {};
    return PQgetvalue(result_.get(), row, col);
}

bool PgResult::is_null(int row, int col) const noexcept {
    return !result_ || PQgetisnull(result_.get(), row, col);
}

std::string_view PgResult::column_name(int col) const noexcept {
    if (!result_) return {};
    const char* name = PQfname(result_.get(), col);
    return name ? name : "";
}

int PgResult::affected_rows() const noexcept {
    if (!result_) return 0;
    const char* val = PQcmdTuples(result_.get());
    if (!val || val[0] == '\0') return 0;
    return std::atoi(val);
}
```

- [ ] **Step 4: pg_connection.cpp 핵심 구현**

```cpp
// apex_shared/lib/adapters/pg/src/pg_connection.cpp

// --- connect_async ---
// 1. PQconnectStart(conninfo) → 비동기 커넥션 시작
// 2. PQstatus(conn) == CONNECTION_BAD → 즉시 에러
// 3. PQsetnonblocking(conn, 1) → non-blocking 모드 설정
// 4. int fd = PQsocket(conn) → fd 획득
// 5. socket_ = make_unique<tcp::socket>(io_ctx_)
//    socket_->assign(tcp::v4(), fd) → Asio IOCP/epoll에 등록
// 6. co_await poll_connect() → PQconnectPoll 루프

// --- poll_connect ---
// loop:
//   PostgresPollingStatusType status = PQconnectPoll(conn_)
//   switch (status):
//     PGRES_POLLING_READING:
//       co_await socket_->async_wait(socket_base::wait_read, use_awaitable)
//     PGRES_POLLING_WRITING:
//       co_await socket_->async_wait(socket_base::wait_write, use_awaitable)
//     PGRES_POLLING_OK:
//       connected_ = true; co_return ok()
//     PGRES_POLLING_FAILED:
//       co_return error(AdapterError)

// --- query_async ---
// 1. PQsendQuery(conn_, sql.data()) → 0이면 에러
// 2. co_return co_await collect_result()

// --- query_params_async ---
// 1. const char* 배열로 params 변환
// 2. PQsendQueryParams(conn_, sql, nParams, nullptr, paramValues, nullptr, nullptr, 0)
// 3. co_return co_await collect_result()

// --- collect_result ---
// loop:
//   co_await socket_->async_wait(socket_base::wait_read, use_awaitable)
//   PQconsumeInput(conn_) → 0이면 에러
//   while (!PQisBusy(conn_)):
//     PGresult* res = PQgetResult(conn_)
//     if (res == nullptr) → 쿼리 완료, 마지막 유효 결과 반환
//     last_result = PgResult(res)
//     if (!last_result.ok()) → 에러 반환

// --- close ---
// 1. release_socket() → socket_->release() (Asio에서 fd 분리, close 안 함)
// 2. PQfinish(conn_) → libpq가 fd close + 리소스 해제
// 3. conn_ = nullptr, connected_ = false

// --- release_socket ---
// socket_->release() — Asio에서 fd를 분리하되 close하지 않음.
// fd의 실제 close는 PQfinish()가 담당.
// 이 순서가 중요: release → PQfinish (역순이면 Asio가 dangling fd 접근)
```

> **Windows 특이사항**:
> - `PQsocket(conn)`은 Windows에서 `SOCKET` 핸들을 반환 (int cast 가능)
> - `tcp::socket::assign(tcp::v4(), (tcp::socket::native_handle_type)PQsocket(conn))`
> - IOCP에 등록되면 `async_wait`가 정상 동작
> - **주의**: Windows IOCP는 한 소켓을 하나의 completion port에만 등록 가능. libpq가 자체적으로 IOCP를 사용하지 않으므로 충돌 없음
> - `socket_->release()` 시 Windows에서는 IOCP 핸들이 해제됨 (CancelIoEx)

```cpp
#pragma once

#include <apex/shared/adapters/pg/pg_connection.hpp>
#include <apex/shared/adapters/pg/pg_config.hpp>
#include <apex/shared/adapters/connection_pool.hpp>

#include <boost/asio/io_context.hpp>

namespace apex::shared::adapters::pg {

/// 코어별 PostgreSQL 커넥션 풀.
/// ConnectionPool CRTP를 상속하여 acquire/release/health_check 로직을 공유받는다.
///
/// 풀 사이즈는 PgBouncer 뒤에 있으므로 작게 유지 (기본 2개/코어).
/// PgBouncer가 실제 PG 커넥션 풀링을 담당.
class PgPool : public ConnectionPool<PgPool> {
public:
    using ConnectionType = PgConnection;

    PgPool(boost::asio::io_context& io_ctx, const PgAdapterConfig& config);

    /// ConnectionPool CRTP 요구사항: 새 커넥션 생성
    /// 동기 버전: PQconnectStart만 호출하고, 실제 비동기 연결은
    /// acquire 후 사용 시점에 lazy로 완료하거나,
    /// 풀 warm-up 시 별도 코루틴에서 connect_async 호출.
    PgConnection do_create_connection();

    /// ConnectionPool CRTP 요구사항: 커넥션 파괴
    void do_destroy_connection(PgConnection& conn);

    /// ConnectionPool CRTP 요구사항: 커넥션 유효성 검증
    bool do_validate(PgConnection& conn);

    /// 비동기 커넥션 생성 + 연결까지 완료 (풀 warm-up용)
    [[nodiscard]] boost::asio::awaitable<apex::core::Result<PgConnection>>
    create_connected();

    /// 풀에서 커넥션을 꺼내되, 연결이 안 된 커넥션이면 connect_async까지 완료.
    /// 일반적인 사용 경로.
    [[nodiscard]] boost::asio::awaitable<apex::core::Result<PgConnection>>
    acquire_connected();

    /// 커넥션 문자열 접근 (디버그/로깅용)
    [[nodiscard]] std::string_view connection_string() const noexcept;

private:
    boost::asio::io_context& io_ctx_;
    PgAdapterConfig config_;
};

} // namespace apex::shared::adapters::pg
```

> **설계 결정 — 동기 create vs 비동기 create**:
> ConnectionPool CRTP의 `do_create_connection()`은 동기 인터페이스. libpq 비동기 커넥션은 코루틴이 필요하므로, `do_create_connection()`에서는 미연결 PgConnection 객체만 생성하고, `acquire_connected()`에서 실제 `connect_async()`를 호출하는 2단계 방식을 사용한다. 이렇게 하면 ConnectionPool의 동기 인터페이스를 깨지 않으면서 비동기 커넥션을 지원할 수 있다.

```cpp
// apex_shared/lib/adapters/pg/src/pg_pool.cpp

PgPool::PgPool(boost::asio::io_context& io_ctx, const PgAdapterConfig& config)
    : ConnectionPool(PoolConfig{
          .min_size = config.pool_size_per_core,
          .max_size = config.pool_size_per_core * 2,  // auto-grow 여유
          .max_idle_time = config.max_idle_time,
          .health_check_interval = config.health_check_interval,
      })
    , io_ctx_(io_ctx)
    , config_(config)
{}

PgConnection PgPool::do_create_connection() {
    // 미연결 PgConnection 생성 (io_context 바인딩만)
    // 실제 커넥션은 acquire_connected()에서 connect_async()로 수립
    return PgConnection(io_ctx_);
}

void PgPool::do_destroy_connection(PgConnection& conn) {
    conn.close();
}

bool PgPool::do_validate(PgConnection& conn) {
    return conn.is_valid();
}

awaitable<Result<PgConnection>> PgPool::create_connected() {
    PgConnection conn(io_ctx_);
    auto result = co_await conn.connect_async(config_.connection_string);
    if (!result.has_value()) {
        co_return std::unexpected(result.error());
    }
    co_return std::move(conn);
}

awaitable<Result<PgConnection>> PgPool::acquire_connected() {
    auto result = acquire();
    if (!result.has_value()) {
        co_return std::unexpected(result.error());
    }
    auto conn = std::move(result.value());
    // 이미 연결된 커넥션이면 바로 반환
    if (conn.is_connected()) {
        co_return std::move(conn);
    }
    // 미연결이면 connect_async 수행
    auto connect_result = co_await conn.connect_async(config_.connection_string);
    if (!connect_result.has_value()) {
        // 연결 실패 — 풀에 반환하지 않고 파괴
        co_return std::unexpected(connect_result.error());
    }
    co_return std::move(conn);
}
```

```cpp
PgAdapter::PgAdapter(PgAdapterConfig config)
    : config_(std::move(config)) {}

PgAdapter::~PgAdapter() = default;

void PgAdapter::do_init(apex::core::CoreEngine& engine) {
    engine_ = &engine;
    pools_.reserve(engine.core_count());
    for (uint32_t i = 0; i < engine.core_count(); ++i) {
        pools_.push_back(std::make_unique<PgPool>(
            engine.io_context(i), config_));
    }
    // Note: 풀 warm-up (미리 커넥션 생성)은 선택사항.
    // PgBouncer 환경에서는 커넥션 생성이 빠르므로 lazy connect로 충분.
}

void PgAdapter::do_drain() {
    // drain 시그널은 AdapterBase가 ready_를 false로 설정.
    // PgPool에 대한 추가 작업 불필요 — 진행 중 쿼리는 완료까지 허용.
}

void PgAdapter::do_close() {
    for (auto& pool : pools_) {
        pool->close_all();
    }
    pools_.clear();
    engine_ = nullptr;
}

PgPool& PgAdapter::current_pool() {
    auto core_id = apex::core::CoreEngine::current_core_id();
    assert(core_id < pools_.size() && "PgAdapter: invalid core_id");
    return *pools_[core_id];
}

awaitable<Result<PgResult>> PgAdapter::query(std::string_view sql) {
    auto conn_result = co_await current_pool().acquire_connected();
    if (!conn_result.has_value()) {
        co_return std::unexpected(conn_result.error());
    }
    auto conn = std::move(conn_result.value());
    auto result = co_await conn.query_async(sql);
    current_pool().release(std::move(conn));
    co_return result;
}

    auto result = co_await conn.query_params_async(sql, params);
    current_pool().release(std::move(conn));
    co_return result;
}
```

## 부록: 기술 레퍼런스

### A. libpq 비동기 API 요약

| 함수 | 역할 | 반환 |
|------|------|------|
| `PQconnectStart(conninfo)` | 비동기 커넥션 시작 | `PGconn*` |
| `PQconnectPoll(conn)` | 커넥션 상태 폴링 | `PostgresPollingStatusType` |
| `PQsocket(conn)` | 소켓 fd 획득 | `int` (Windows: `SOCKET`) |
| `PQsetnonblocking(conn, 1)` | non-blocking 모드 설정 | `0` = 성공 |
| `PQstatus(conn)` | 커넥션 상태 확인 | `ConnStatusType` |
| `PQsendQuery(conn, sql)` | 비동기 쿼리 전송 | `1` = 성공 |
| `PQsendQueryParams(...)` | 비동기 파라미터 쿼리 | `1` = 성공 |
| `PQconsumeInput(conn)` | 소켓에서 데이터 읽기 | `1` = 성공 |
| `PQisBusy(conn)` | 결과 준비 여부 | `0` = 준비됨 |
| `PQgetResult(conn)` | 결과 수신 | `PGresult*` (NULL = 완료) |
| `PQfinish(conn)` | 커넥션 종료 + fd 닫기 | void |

### B. Asio 소켓 assign 패턴

```cpp
// Windows (IOCP)
int fd = PQsocket(conn);
boost::asio::ip::tcp::socket socket(io_ctx);
socket.assign(boost::asio::ip::tcp::v4(),
              static_cast<boost::asio::ip::tcp::socket::native_handle_type>(fd));

// async_wait로 readable/writable 이벤트 대기
co_await socket.async_wait(
    boost::asio::socket_base::wait_read,
    boost::asio::use_awaitable);
```

**주의사항**:
- `socket.close()` 대신 `socket.release()` 사용 — fd는 libpq가 관리
- `PQfinish()` 전에 반드시 `socket.release()` 호출
- Windows IOCP: 소켓당 하나의 completion port만 가능 (libpq는 IOCP 미사용이므로 충돌 없음)

### C. PgBouncer 환경

```
어댑터 → PgBouncer(6432) → PostgreSQL(5432)
```

- PgBouncer가 서버 측 커넥션 풀링 담당
- 어댑터 풀 사이즈는 작게 유지 (코어당 2~4개)
- PgBouncer의 `pool_mode = transaction` 권장 (쿼리당 커넥션 재사용)
- 모든 환경(개발/테스트/프로덕션)에서 PgBouncer 사용으로 환경 일관성 보장

### D. 에러 처리 흐름

```
libpq 에러 → PgConnection에서 감지
  → ErrorCode::AdapterError 반환
  → 서비스에서 Result<PgResult> 패턴 매칭:
      if (result.has_value()) { /* 성공 */ }
      else { /* result.error() == ErrorCode::AdapterError */ }
```

상세 에러 정보가 필요한 경우 `AdapterError` 구조체를 통해 libpq 원본 에러코드 + 메시지 전달 (향후 Result의 에러 타입을 `ErrorCode`에서 `AdapterError`로 확장 가능).
