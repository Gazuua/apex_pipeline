# BACKLOG-137 + BACKLOG-151 설계 부채 해소 구현 계획

**목표**: KafkaConsumer 소멸자 handler lifetime 보호 + FrameError/ErrorCode 에러 타입 통일

**아키텍처**: 두 작업은 파일 범위가 완전히 독립적 (137=kafka 어댑터, 151=코어 프레임 코덱). 병렬 실행 가능.

---

## Task 1: BACKLOG-137 — KafkaConsumer shared_ptr + weak_ptr

### 변경 파일
- `apex_shared/lib/adapters/kafka/include/apex/shared/adapters/kafka/kafka_consumer.hpp` — enable_shared_from_this 상속 추가
- `apex_shared/lib/adapters/kafka/include/apex/shared/adapters/kafka/kafka_adapter.hpp` — vector 타입 변경
- `apex_shared/lib/adapters/kafka/src/kafka_adapter.cpp` — make_unique → make_shared
- `apex_shared/lib/adapters/kafka/src/kafka_consumer.cpp` — async handler에 weak_ptr 캡처

### 구현 상세

**kafka_consumer.hpp:**
- `class KafkaConsumer` → `class KafkaConsumer : public std::enable_shared_from_this<KafkaConsumer>`
- `#include <memory>` 이미 있음

**kafka_adapter.hpp:**
- `std::vector<std::unique_ptr<KafkaConsumer>> consumers_;` → `std::vector<std::shared_ptr<KafkaConsumer>> consumers_;`

**kafka_adapter.cpp:**
- `std::make_unique<KafkaConsumer>(...)` → `std::make_shared<KafkaConsumer>(...)`
- 소멸자 변경 없음 (consumers_.clear()가 shared_ptr 해제, 미완료 handler의 weak_ptr은 lock 실패)

**kafka_consumer.cpp — schedule_async_wait():**
```cpp
void KafkaConsumer::schedule_async_wait()
{
#ifndef _WIN32
    if (!consuming_ || !pipe_desc_)
        return;

    std::weak_ptr<KafkaConsumer> weak_self = shared_from_this();
    pipe_desc_->async_wait(boost::asio::posix::stream_descriptor::wait_read,
                           [weak_self](const boost::system::error_code& ec) {
                               auto self = weak_self.lock();
                               if (!self || ec || !self->consuming_)
                                   return;
                               char buf[64];
                               (void)::read(self->pipe_fds_[0], buf, sizeof(buf));
                               self->poll_messages();
                               self->schedule_async_wait();
                           });
#endif
}
```

**kafka_consumer.cpp — schedule_timer_poll():**
```cpp
void KafkaConsumer::schedule_timer_poll()
{
    if (!consuming_)
        return;
    if (!poll_timer_)
        poll_timer_ = std::make_unique<boost::asio::steady_timer>(io_ctx_);
    poll_timer_->expires_after(config_.consumer_poll_interval);
    std::weak_ptr<KafkaConsumer> weak_self = shared_from_this();
    poll_timer_->async_wait([weak_self](const boost::system::error_code& ec) {
        auto self = weak_self.lock();
        if (!self || ec || !self->consuming_)
            return;
        self->poll_messages();
        self->schedule_timer_poll();
    });
}
```

- 기존 84/84 테스트 통과 확인 (API 변경 없으므로 기존 테스트 호환)
- KafkaAdapter 관련 mock 테스트: `apex_shared/tests/unit/test_kafka_adapter.cpp`, `test_kafka_consumer.cpp`

## Task 2: BACKLOG-151 — FrameError 제거 + ErrorCode 통일

### 변경 파일
- `apex_core/include/apex/core/error_code.hpp` — BodyTooLarge 추가
- `apex_core/include/apex/core/frame_codec.hpp` — FrameError enum 삭제, 반환 타입 변경
- `apex_core/src/frame_codec.cpp` — ErrorCode 반환으로 전환
- `apex_shared/lib/protocols/tcp/include/apex/shared/protocols/tcp/tcp_binary_protocol.hpp` — 단순 포워딩으로 축소
- `apex_core/tests/test_mocks.hpp` — MockProtocol 변환 switch 제거
- `apex_core/tests/unit/test_frame_codec.cpp` — FrameError → ErrorCode assertion 변경

### 구현 상세

**error_code.hpp:**
- `CrossCoreFuncException = 17,` 다음에 `BodyTooLarge = 18,` 추가
- `error_code_name` switch에 `case ErrorCode::BodyTooLarge: return "BodyTooLarge";` 추가

**frame_codec.hpp:**
- `enum class FrameError` 삭제
- `[[nodiscard]] static std::expected<Frame, FrameError> try_decode(RingBuffer& buf);`
  → `[[nodiscard]] static Result<Frame> try_decode(RingBuffer& buf);`
- `#include <apex/core/result.hpp>` 추가 (Result 타입 사용)

**frame_codec.cpp:**
- `FrameError::InsufficientData` → `ErrorCode::InsufficientData`
- `FrameError::BodyTooLarge` → `ErrorCode::BodyTooLarge`
- `FrameError::UnsupportedProtocolVersion` → `ErrorCode::UnsupportedProtocolVersion`
- `FrameError::HeaderParseError` → `ErrorCode::InvalidMessage`
- include: `#include <apex/core/error_code.hpp>` 추가 (기존에 없으면)

**tcp_binary_protocol.hpp:**
```cpp
[[nodiscard]] static apex::core::Result<Frame> try_decode(apex::core::RingBuffer& buf)

    return apex::core::FrameCodec::try_decode(buf);

```

**test_mocks.hpp MockProtocol:**
- FrameError→ErrorCode 변환 switch 제거, FrameCodec::try_decode 직접 반환

**test_frame_codec.cpp:**
- `FrameError::InsufficientData` → `apex::core::ErrorCode::InsufficientData`
- `FrameError::BodyTooLarge` → `apex::core::ErrorCode::BodyTooLarge`
