# v0.4.1 Kafka 어댑터 — Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** librdkafka 2.x 기반의 Kafka Producer/Consumer/KafkaSink를 구현하여 Asio 이벤트 루프에 통합한다. Producer는 전역 공유 1개 인스턴스로 모든 코어에서 직접 호출하며, Consumer는 파티션:코어 매핑으로 코어별 독립 소비한다. KafkaSink는 spdlog sink으로 Kafka에 구조화 로그를 전송한다.

**Architecture:** KafkaAdapter는 AdapterBase CRTP를 상속하여 Server에 인프라 컴포넌트로 등록된다. Producer는 `rd_kafka_produce()` 스레드세이프 특성을 이용해 SPSC 큐 없이 코어에서 직접 호출한다. Consumer는 `rd_kafka_queue_io_event_enable()`로 pipe fd를 Asio에 등록하되 (Linux), Windows에서는 steady_timer 5ms 폴링으로 fallback한다. delivery callback은 Asio로 브릿지하여 비동기 결과를 코루틴에 전달한다.

**Tech Stack:** C++23, Boost.Asio, librdkafka 2.x (vcpkg), spdlog, Google Test, CMake

**Spec:** `docs/apex_common/plans/20260313_063601_v0.4_external_adapters_design.md` (§5 Kafka 어댑터)

**Depends on:** Plan 1 (v0.4.0.1 공통 추상화 — AdapterBase, AdapterError, ConnectionPool, Server add_adapter API)

- [ ] **Step 3: kafka_producer.hpp 작성**

```cpp
// apex_shared/lib/adapters/kafka/include/apex/shared/adapters/kafka/kafka_producer.hpp
#pragma once

#include <apex/shared/adapters/kafka/kafka_config.hpp>
#include <apex/shared/adapters/adapter_error.hpp>
#include <apex/core/result.hpp>

#include <boost/asio/any_io_executor.hpp>
#include <boost/asio/awaitable.hpp>
#include <boost/asio/steady_timer.hpp>

#include <librdkafka/rdkafka.h>

#include <cstdint>
#include <functional>
#include <memory>
#include <span>
#include <string>
#include <string_view>

namespace apex::shared::adapters::kafka {

/// 전역 공유 Kafka Producer.
///
/// rd_kafka_produce()가 스레드세이프이므로 모든 코어에서 직접 호출 가능.
/// SPSC 큐 불필요 — librdkafka 내부 큐가 배치 처리를 수행.
///
/// Delivery callback은 librdkafka의 내부 스레드에서 호출되므로,
/// 결과를 Asio executor로 포스트하여 코루틴에 전달한다.
///
/// 사용 패턴:
///   co_await producer.produce("topic", key, payload);  // fire-and-forget
///   auto result = co_await producer.produce_and_wait("topic", key, payload);
class KafkaProducer {
public:
    /// @param config Kafka 설정
    /// @param poll_executor rd_kafka_poll()을 실행할 executor (보통 control_io)
    explicit KafkaProducer(const KafkaConfig& config);
    ~KafkaProducer();

    KafkaProducer(const KafkaProducer&) = delete;
    KafkaProducer& operator=(const KafkaProducer&) = delete;

    /// Producer 초기화. rd_kafka_new() + 설정 적용.
    /// @return 실패 시 AdapterError (브로커 연결 실패 등)
    [[nodiscard]] apex::core::Result<void> init();

    /// Fire-and-forget 프로듀스. 내부 큐에 enqueue만 하고 즉시 리턴.
    /// delivery callback에서 에러 로깅.
    /// @param topic 토픽 이름
    /// @param key 메시지 키 (파티셔닝 기준, 빈 값 허용)
    /// @param payload 메시지 바디
    [[nodiscard]] apex::core::Result<void> produce(
        std::string_view topic,
        std::string_view key,
        std::span<const uint8_t> payload);

    /// 오버로드: string payload
    [[nodiscard]] apex::core::Result<void> produce(
        std::string_view topic,
        std::string_view key,
        std::string_view payload);

    /// librdkafka 내부 큐 폴링 (delivery callback 처리).
    /// 주기적으로 호출해야 callback이 fire된다.
    /// KafkaAdapter::do_init()에서 타이머로 스케줄링.
    /// @param timeout_ms 폴링 타임아웃 (0 = non-blocking)
    /// @return 처리된 이벤트 수
    int poll(int timeout_ms = 0);

    /// 큐에 남은 메시지를 모두 flush (graceful shutdown용).
    /// @param timeout 최대 대기 시간
    /// @return flush 완료 여부
    [[nodiscard]] bool flush(std::chrono::milliseconds timeout);

    /// 현재 큐에 대기 중인 메시지 수
    [[nodiscard]] int32_t outq_len() const noexcept;

    /// 초기화 완료 여부
    [[nodiscard]] bool initialized() const noexcept { return rk_ != nullptr; }

    /// 통계: 전송 성공/실패 카운터
    [[nodiscard]] uint64_t total_produced() const noexcept { return total_produced_; }
    [[nodiscard]] uint64_t total_failed() const noexcept { return total_failed_; }

private:
    /// librdkafka delivery report callback (static, C 콜백)
    static void delivery_report_cb(rd_kafka_t* rk,
                                   const rd_kafka_message_t* msg,
                                   void* opaque);

    /// rd_kafka_topic_t 캐시 (토픽별)
    rd_kafka_topic_t* get_or_create_topic(std::string_view topic);

    KafkaConfig config_;
    rd_kafka_t* rk_ = nullptr;                          ///< librdkafka 핸들

    /// 토픽 핸들 캐시 (rd_kafka_topic_new는 비교적 무거움)
    struct TopicEntry {
        std::string name;
        rd_kafka_topic_t* rkt = nullptr;
    };
    std::vector<TopicEntry> topic_cache_;

    // 통계 (atomic — 멀티코어에서 동시 produce 가능)
    std::atomic<uint64_t> total_produced_{0};
    std::atomic<uint64_t> total_failed_{0};
};

} // namespace apex::shared::adapters::kafka
```

- [ ] **Step 4: kafka_producer.cpp 구현**

핵심 구현 포인트:

```cpp
// apex_shared/lib/adapters/kafka/src/kafka_producer.cpp
#include <apex/shared/adapters/kafka/kafka_producer.hpp>

#include <spdlog/spdlog.h>

#include <cstring>

namespace apex::shared::adapters::kafka {

KafkaProducer::KafkaProducer(const KafkaConfig& config)
    : config_(config) {}

KafkaProducer::~KafkaProducer() {
    if (rk_) {
        // 토픽 핸들 해제
        for (auto& entry : topic_cache_) {
            if (entry.rkt) rd_kafka_topic_destroy(entry.rkt);
        }
        topic_cache_.clear();

        // flush 후 핸들 해제
        rd_kafka_flush(rk_, 5000);
        rd_kafka_destroy(rk_);
        rk_ = nullptr;
    }
}

apex::core::Result<void> KafkaProducer::init() {
    char errstr[512];
    rd_kafka_conf_t* conf = rd_kafka_conf_new();

    // 브로커
    rd_kafka_conf_set(conf, "bootstrap.servers", config_.brokers.c_str(),
                      errstr, sizeof(errstr));

    // client.id
    rd_kafka_conf_set(conf, "client.id", config_.client_id.c_str(),
                      errstr, sizeof(errstr));

    // Producer 배치 설정
    rd_kafka_conf_set(conf, "linger.ms",
                      std::to_string(config_.producer_batch_ms.count()).c_str(),
                      errstr, sizeof(errstr));
    rd_kafka_conf_set(conf, "batch.size",
                      std::to_string(config_.producer_batch_size).c_str(),
                      errstr, sizeof(errstr));
    rd_kafka_conf_set(conf, "compression.type",
                      config_.compression_type.c_str(),
                      errstr, sizeof(errstr));
    rd_kafka_conf_set(conf, "queue.buffering.max.messages",
                      std::to_string(config_.queue_buffering_max_messages).c_str(),
                      errstr, sizeof(errstr));

    // Delivery report callback
    rd_kafka_conf_set_dr_msg_cb(conf, &KafkaProducer::delivery_report_cb);
    rd_kafka_conf_set_opaque(conf, this);

    // Producer 생성
    rk_ = rd_kafka_new(RD_KAFKA_PRODUCER, conf, errstr, sizeof(errstr));
    if (!rk_) {
        // conf는 rd_kafka_new 실패 시에도 소유권이 이전되지 않음
        // — 아님, 실패해도 conf는 rd_kafka_new가 소유. 문서 확인 필요.
        // 안전하게: rd_kafka_new 실패 시 conf는 이미 destroyed.
        spdlog::error("KafkaProducer::init failed: {}", errstr);
        return std::unexpected(apex::core::ErrorCode::AdapterError);
    }

    spdlog::info("KafkaProducer initialized: brokers={}", config_.brokers);
    return {};
}

apex::core::Result<void> KafkaProducer::produce(
    std::string_view topic,
    std::string_view key,
    std::span<const uint8_t> payload)
{
    if (!rk_) {
        return std::unexpected(apex::core::ErrorCode::AdapterError);
    }

    auto* rkt = get_or_create_topic(topic);
    if (!rkt) {
        return std::unexpected(apex::core::ErrorCode::AdapterError);
    }

    int err = rd_kafka_produce(
        rkt,
        RD_KAFKA_PARTITION_UA,          // 자동 파티셔닝
        RD_KAFKA_MSG_F_COPY,            // payload 복사
        const_cast<void*>(static_cast<const void*>(payload.data())),
        payload.size(),
        key.empty() ? nullptr : key.data(),
        key.size(),
        nullptr                          // opaque (per-message)
    );

    if (err == -1) {
        rd_kafka_resp_err_t last_err = rd_kafka_last_error();
        spdlog::warn("KafkaProducer::produce failed: topic={}, err={}",
                      topic, rd_kafka_err2str(last_err));
        total_failed_.fetch_add(1, std::memory_order_relaxed);
        return std::unexpected(apex::core::ErrorCode::AdapterError);
    }

    return {};
}

apex::core::Result<void> KafkaProducer::produce(
    std::string_view topic,
    std::string_view key,
    std::string_view payload)
{
    return produce(topic, key,
        std::span<const uint8_t>(
            reinterpret_cast<const uint8_t*>(payload.data()),
            payload.size()));
}

int KafkaProducer::poll(int timeout_ms) {
    if (!rk_) return 0;
    return rd_kafka_poll(rk_, timeout_ms);
}

bool KafkaProducer::flush(std::chrono::milliseconds timeout) {
    if (!rk_) return true;
    rd_kafka_resp_err_t err = rd_kafka_flush(rk_,
        static_cast<int>(timeout.count()));
    return err == RD_KAFKA_RESP_ERR_NO_ERROR;
}

int32_t KafkaProducer::outq_len() const noexcept {
    if (!rk_) return 0;
    return rd_kafka_outq_len(rk_);
}

void KafkaProducer::delivery_report_cb(
    rd_kafka_t* /*rk*/,
    const rd_kafka_message_t* msg,
    void* opaque)
{
    auto* self = static_cast<KafkaProducer*>(opaque);
    if (msg->err) {
        spdlog::warn("Kafka delivery failed: topic={}, err={}",
                      rd_kafka_topic_name(msg->rkt),
                      rd_kafka_err2str(msg->err));
        self->total_failed_.fetch_add(1, std::memory_order_relaxed);
    } else {
        self->total_produced_.fetch_add(1, std::memory_order_relaxed);
    }
}

rd_kafka_topic_t* KafkaProducer::get_or_create_topic(std::string_view topic) {
    // 캐시 검색
    for (auto& entry : topic_cache_) {
        if (entry.name == topic) return entry.rkt;
    }
    // 새 토픽 핸들 생성
    rd_kafka_topic_t* rkt = rd_kafka_topic_new(
        rk_, std::string(topic).c_str(), nullptr);
    if (rkt) {
        topic_cache_.push_back({std::string(topic), rkt});
    }
    return rkt;
}

} // namespace apex::shared::adapters::kafka
```

```cpp
#include <apex/shared/adapters/kafka/kafka_config.hpp>
#include <apex/shared/adapters/adapter_error.hpp>
#include <apex/core/result.hpp>

#include <boost/asio/io_context.hpp>
#include <boost/asio/steady_timer.hpp>
#ifndef _WIN32
#include <boost/asio/posix/stream_descriptor.hpp>
#endif

#include <librdkafka/rdkafka.h>

#include <functional>
#include <memory>
#include <span>

namespace apex::shared::adapters::kafka {

/// 수신 메시지 콜백 시그니처.
/// @param topic 토픽 이름
/// @param partition 파티션 번호
/// @param key 메시지 키
/// @param payload 메시지 바디
/// @param offset 카프카 오프셋
using MessageCallback = std::function<void(
    std::string_view topic,
    int32_t partition,
    std::span<const uint8_t> key,
    std::span<const uint8_t> payload,
    int64_t offset)>;

/// 코어별 Kafka Consumer 인스턴스.
///
/// 파티션:코어 매핑으로 동작:
/// - Consumer Group이 파티션을 분배
/// - 각 코어 스레드의 io_context에 통합
///
/// Asio 통합 방식:
/// - Linux: rd_kafka_queue_io_event_enable() → pipe fd → Asio stream_descriptor
/// - Windows: steady_timer 기반 주기적 폴링 (5ms 기본)
class KafkaConsumer {
public:
    /// @param config Kafka 설정
    /// @param core_id 이 Consumer가 바인딩되는 코어 ID
    /// @param io_ctx 해당 코어의 io_context
    explicit KafkaConsumer(const KafkaConfig& config,
                           uint32_t core_id,
                           boost::asio::io_context& io_ctx);
    ~KafkaConsumer();

    KafkaConsumer(const KafkaConsumer&) = delete;
    KafkaConsumer& operator=(const KafkaConsumer&) = delete;

    /// Consumer 초기화 + 토픽 구독.
    /// group.id에 core_id를 suffix로 붙이지 않음 —
    /// 같은 consumer group 내에서 파티션 자동 분배.
    [[nodiscard]] apex::core::Result<void> init();

    /// 메시지 수신 콜백 등록. init() 전에 호출.
    void set_message_callback(MessageCallback cb);

    /// Asio 이벤트 루프에 소비 스케줄링 시작.
    /// Linux: pipe fd → async_wait
    /// Windows: steady_timer 폴링
    void start_consuming();

    /// 소비 중지 (shutdown 시).
    void stop_consuming();

    /// 초기화 완료 여부
    [[nodiscard]] bool initialized() const noexcept { return rk_ != nullptr; }

    /// 소비 중 여부
    [[nodiscard]] bool consuming() const noexcept { return consuming_; }

    /// 통계
    [[nodiscard]] uint64_t total_consumed() const noexcept { return total_consumed_; }

private:
    /// 메시지 폴링 + 콜백 호출 (non-blocking)
    void poll_messages();

    /// Linux: pipe fd 이벤트 대기 → 메시지 폴링 루프
    void schedule_async_wait();

    /// Windows: 타이머 기반 폴링 루프
    void schedule_timer_poll();

    KafkaConfig config_;
    uint32_t core_id_;
    boost::asio::io_context& io_ctx_;

    rd_kafka_t* rk_ = nullptr;
    rd_kafka_queue_t* rkqu_ = nullptr;          ///< consumer queue handle

    MessageCallback message_cb_;
    bool consuming_ = false;
    uint64_t total_consumed_ = 0;

#ifndef _WIN32
    // Linux: pipe fd → Asio stream_descriptor
    int pipe_fds_[2] = {-1, -1};
    std::unique_ptr<boost::asio::posix::stream_descriptor> pipe_desc_;
#endif

    // Windows (또는 fallback): 타이머 폴링
    std::unique_ptr<boost::asio::steady_timer> poll_timer_;
};

} // namespace apex::shared::adapters::kafka
```

```cpp
// apex_shared/lib/adapters/kafka/src/kafka_consumer.cpp
#include <apex/shared/adapters/kafka/kafka_consumer.hpp>

#include <spdlog/spdlog.h>

#ifndef _WIN32
#include <unistd.h>  // pipe(), read()
#endif

namespace apex::shared::adapters::kafka {

KafkaConsumer::KafkaConsumer(const KafkaConfig& config,
                             uint32_t core_id,
                             boost::asio::io_context& io_ctx)
    : config_(config), core_id_(core_id), io_ctx_(io_ctx) {}

KafkaConsumer::~KafkaConsumer() {
    stop_consuming();
    if (rk_) {
        rd_kafka_consumer_close(rk_);
#ifndef _WIN32
        if (pipe_fds_[0] != -1) { close(pipe_fds_[0]); close(pipe_fds_[1]); }
#endif
        rd_kafka_queue_destroy(rkqu_);
        rd_kafka_destroy(rk_);
        rk_ = nullptr;
    }
}

apex::core::Result<void> KafkaConsumer::init() {
    char errstr[512];
    rd_kafka_conf_t* conf = rd_kafka_conf_new();

    rd_kafka_conf_set(conf, "bootstrap.servers", config_.brokers.c_str(),
                      errstr, sizeof(errstr));
    rd_kafka_conf_set(conf, "group.id", config_.consumer_group.c_str(),
                      errstr, sizeof(errstr));
    rd_kafka_conf_set(conf, "client.id",
                      (config_.client_id + "-consumer-" + std::to_string(core_id_)).c_str(),
                      errstr, sizeof(errstr));
    // auto.offset.reset = earliest (신규 consumer group일 때)
    rd_kafka_conf_set(conf, "auto.offset.reset", "earliest",
                      errstr, sizeof(errstr));
    // enable.auto.commit = true (기본값, 편의상 유지)
    rd_kafka_conf_set(conf, "enable.auto.commit", "true",
                      errstr, sizeof(errstr));

    rk_ = rd_kafka_new(RD_KAFKA_CONSUMER, conf, errstr, sizeof(errstr));
    if (!rk_) {
        spdlog::error("KafkaConsumer[core={}] init failed: {}", core_id_, errstr);
        return std::unexpected(apex::core::ErrorCode::AdapterError);
    }

    // consumer queue 획득
    rd_kafka_poll_set_consumer(rk_);
    rkqu_ = rd_kafka_queue_get_consumer(rk_);

    // 토픽 구독
    if (!config_.consume_topics.empty()) {
        rd_kafka_topic_partition_list_t* topics =
            rd_kafka_topic_partition_list_new(
                static_cast<int>(config_.consume_topics.size()));
        for (const auto& t : config_.consume_topics) {
            rd_kafka_topic_partition_list_add(topics, t.c_str(),
                                              RD_KAFKA_PARTITION_UA);
        }
        rd_kafka_resp_err_t err = rd_kafka_subscribe(rk_, topics);
        rd_kafka_topic_partition_list_destroy(topics);

        if (err != RD_KAFKA_RESP_ERR_NO_ERROR) {
            spdlog::error("KafkaConsumer[core={}] subscribe failed: {}",
                          core_id_, rd_kafka_err2str(err));
            return std::unexpected(apex::core::ErrorCode::AdapterError);
        }
    }

#ifndef _WIN32
    // Linux: pipe fd → rd_kafka_queue_io_event_enable
    if (pipe(pipe_fds_) == -1) {
        spdlog::error("KafkaConsumer[core={}] pipe() failed", core_id_);
        return std::unexpected(apex::core::ErrorCode::AdapterError);
    }
    rd_kafka_queue_io_event_enable(rkqu_, pipe_fds_[1], "1", 1);
    pipe_desc_ = std::make_unique<boost::asio::posix::stream_descriptor>(
        io_ctx_, pipe_fds_[0]);
#endif

    spdlog::info("KafkaConsumer[core={}] initialized: group={}",
                  core_id_, config_.consumer_group);
    return {};
}

void KafkaConsumer::set_message_callback(MessageCallback cb) {
    message_cb_ = std::move(cb);
}

void KafkaConsumer::start_consuming() {
    consuming_ = true;
#ifndef _WIN32
    if (pipe_desc_) {
        schedule_async_wait();
    } else {
        schedule_timer_poll();
    }
#else
    schedule_timer_poll();
#endif
}

void KafkaConsumer::stop_consuming() {
    consuming_ = false;
#ifndef _WIN32
    if (pipe_desc_) pipe_desc_->cancel();
#endif
    if (poll_timer_) poll_timer_->cancel();
}

void KafkaConsumer::poll_messages() {
    if (!rk_ || !consuming_) return;

    // Non-blocking batch poll
    constexpr int max_batch = 64;
    for (int i = 0; i < max_batch; ++i) {
        rd_kafka_message_t* msg = rd_kafka_consumer_poll(rk_, 0);
        if (!msg) break;

        if (msg->err == RD_KAFKA_RESP_ERR_NO_ERROR) {
            if (message_cb_) {
                std::span<const uint8_t> key_span;
                if (msg->key && msg->key_len > 0) {
                    key_span = {static_cast<const uint8_t*>(msg->key),
                                msg->key_len};
                }
                std::span<const uint8_t> payload_span;
                if (msg->payload && msg->len > 0) {
                    payload_span = {static_cast<const uint8_t*>(msg->payload),
                                    msg->len};
                }
                message_cb_(rd_kafka_topic_name(msg->rkt),
                             msg->partition,
                             key_span, payload_span,
                             msg->offset);
            }
            ++total_consumed_;
        } else if (msg->err != RD_KAFKA_RESP_ERR__PARTITION_EOF) {
            spdlog::warn("KafkaConsumer[core={}] poll error: {}",
                          core_id_, rd_kafka_message_errstr(msg));
        }
        rd_kafka_message_destroy(msg);
    }
}

void KafkaConsumer::schedule_async_wait() {
#ifndef _WIN32
    if (!consuming_ || !pipe_desc_) return;

    pipe_desc_->async_wait(
        boost::asio::posix::stream_descriptor::wait_read,
        [this](const boost::system::error_code& ec) {
            if (ec || !consuming_) return;

            // pipe에서 이벤트 바이트 소비
            char buf[64];
            (void)::read(pipe_fds_[0], buf, sizeof(buf));

            poll_messages();

            // 다음 이벤트 대기
            schedule_async_wait();
        });
#endif
}

void KafkaConsumer::schedule_timer_poll() {
    if (!consuming_) return;

    if (!poll_timer_) {
        poll_timer_ = std::make_unique<boost::asio::steady_timer>(io_ctx_);
    }
    poll_timer_->expires_after(config_.consumer_poll_interval);
    poll_timer_->async_wait([this](const boost::system::error_code& ec) {
        if (ec || !consuming_) return;
        poll_messages();
        schedule_timer_poll();
    });
}

} // namespace apex::shared::adapters::kafka
```

```cpp
/// Kafka 어댑터. AdapterBase CRTP 상속.
///
/// 전역 1개 인스턴스로 Server에 등록되며, 내부적으로:
/// - KafkaProducer 1개 (전역 공유, 스레드세이프)
/// - KafkaConsumer N개 (코어별 1개, 파티션:코어 매핑)

/// 사용:
///   Server({.port = 9000, .num_cores = 4})
///       .add_adapter<KafkaAdapter>(kafka_config)
///       .add_service<MyService>()
///       .run();
///
///   // 서비스에서:
///   auto& kafka = server.adapter<KafkaAdapter>();
///   kafka.produce("topic", key, payload);
class KafkaAdapter : public apex::shared::adapters::AdapterBase<KafkaAdapter> {
public:
    explicit KafkaAdapter(KafkaConfig config);
    ~KafkaAdapter();

    // --- AdapterBase CRTP 인터페이스 ---

    /// 코어별 리소스 초기화.
    /// Producer init + 코어별 Consumer 생성/init + poll 타이머 시작.
    void do_init(apex::core::CoreEngine& engine);

    /// 새 produce 요청 거부, Consumer 소비 중지.
    void do_drain();

    /// Producer flush + Consumer close + 리소스 정리.
    void do_close();

    /// 어댑터 이름
    [[nodiscard]] std::string_view do_name() const noexcept { return "kafka"; }

    // --- Kafka API (서비스에서 사용) ---

    /// Fire-and-forget 프로듀스.
    [[nodiscard]] apex::core::Result<void> produce(
        std::string_view topic,
        std::string_view key,
        std::span<const uint8_t> payload);
```

```cpp
#include <apex/shared/adapters/kafka/kafka_adapter.hpp>

#include <spdlog/spdlog.h>

#include <cassert>
#include <chrono>

namespace apex::shared::adapters::kafka {

KafkaAdapter::KafkaAdapter(KafkaConfig config)
    : config_(std::move(config))
    , producer_(std::make_unique<KafkaProducer>(config_)) {}

KafkaAdapter::~KafkaAdapter() = default;

void KafkaAdapter::do_init(apex::core::CoreEngine& engine) {
    // 1. Producer 초기화
    auto result = producer_->init();
    if (!result.has_value()) {
        spdlog::error("KafkaAdapter: Producer init failed");
        return;  // TODO: 에러 전파 전략 결정
    }

    // 2. 코어별 Consumer 생성 + 초기화
    uint32_t num_cores = engine.core_count();
    consumers_.reserve(num_cores);
    for (uint32_t i = 0; i < num_cores; ++i) {
        auto consumer = std::make_unique<KafkaConsumer>(
            config_, i, engine.io_context(i));
        if (message_cb_) {
            consumer->set_message_callback(message_cb_);
        }
        auto init_result = consumer->init();
        if (!init_result.has_value()) {
            spdlog::error("KafkaAdapter: Consumer[core={}] init failed", i);
        }
        consumers_.push_back(std::move(consumer));
    }

    // 3. Consumer 소비 시작
    for (auto& consumer : consumers_) {
        if (consumer->initialized()) {
            consumer->start_consuming();
        }
    }

    // 4. Producer poll timer 시작 (코어 0의 io_context 사용)
    start_producer_poll_timer(engine.io_context(0));

    spdlog::info("KafkaAdapter initialized: {} cores, brokers={}",
                  num_cores, config_.brokers);
}

void KafkaAdapter::do_drain() {
    draining_ = true;
    // Consumer 소비 중지
    for (auto& consumer : consumers_) {
        consumer->stop_consuming();
    }
    spdlog::info("KafkaAdapter: drain started");
}

void KafkaAdapter::do_close() {
    // Producer poll timer 중지
    if (producer_poll_timer_) {
        producer_poll_timer_->cancel();
    }

    // Producer flush (최대 10초 대기)
    if (producer_->initialized()) {
        spdlog::info("KafkaAdapter: flushing producer (outq={})",
                      producer_->outq_len());
        producer_->flush(std::chrono::seconds{10});
    }

    // Consumer 정리
    consumers_.clear();

    // Producer 정리 (소멸자에서 flush + destroy)
    producer_.reset();

    spdlog::info("KafkaAdapter: closed");
}

apex::core::Result<void> KafkaAdapter::produce(
    std::string_view topic,
    std::string_view key,
    std::span<const uint8_t> payload)
{
    if (draining_) {
        return std::unexpected(apex::core::ErrorCode::AdapterError);
    }
    return producer_->produce(topic, key, payload);
}

apex::core::Result<void> KafkaAdapter::produce(
    std::string_view topic,
    std::string_view key,
    std::string_view payload)
{
    if (draining_) {
        return std::unexpected(apex::core::ErrorCode::AdapterError);
    }
    return producer_->produce(topic, key, payload);
}

void KafkaAdapter::set_message_callback(MessageCallback cb) {
    message_cb_ = std::move(cb);
}

KafkaConsumer& KafkaAdapter::consumer(uint32_t core_id) {
    assert(core_id < consumers_.size() && "core_id out of range");
    return *consumers_[core_id];
}

void KafkaAdapter::start_producer_poll_timer(boost::asio::io_context& io_ctx) {
    producer_poll_timer_ = std::make_unique<boost::asio::steady_timer>(io_ctx);
    on_producer_poll_tick();
}

void KafkaAdapter::on_producer_poll_tick() {
    if (!producer_poll_timer_ || draining_) return;

    // Producer delivery callback 처리
    producer_->poll(0);

    // 100ms 주기로 재스케줄 (delivery report 처리 지연 ≤ 100ms)
    producer_poll_timer_->expires_after(std::chrono::milliseconds{100});
    producer_poll_timer_->async_wait([this](const boost::system::error_code& ec) {
        if (ec) return;
        on_producer_poll_tick();
    });
}
```

```cpp
#include <apex/shared/adapters/kafka/kafka_sink.hpp>

#include <spdlog/spdlog.h>
#include <spdlog/pattern_formatter.h>

#include <chrono>
#include <iomanip>
#include <sstream>

namespace apex::shared::adapters::kafka {

KafkaSink::KafkaSink(KafkaProducer& producer, std::string topic)
    : producer_(producer), topic_(std::move(topic)) {}

void KafkaSink::sink_it_(const spdlog::details::log_msg& msg) {
    auto json = format_json(msg);
    // fire-and-forget — 에러는 delivery callback에서 로깅
    (void)producer_.produce(topic_, "", std::string_view(json));
}

void KafkaSink::flush_() {
    producer_.poll(0);
}

std::string KafkaSink::format_json(const spdlog::details::log_msg& msg) const {
    // ISO 8601 타임스탬프
    auto time_t = std::chrono::system_clock::to_time_t(msg.time);
    auto ms = std::chrono::duration_cast<std::chrono::milliseconds>(
        msg.time.time_since_epoch()) % 1000;

    std::ostringstream oss;
    oss << "{\"ts\":\"";

    // UTC 시간
    std::tm tm_buf{};
#ifdef _WIN32
    gmtime_s(&tm_buf, &time_t);
#else
    gmtime_r(&time_t, &tm_buf);
#endif
    oss << std::put_time(&tm_buf, "%Y-%m-%dT%H:%M:%S");
    oss << '.' << std::setfill('0') << std::setw(3) << ms.count() << "Z\"";

    // level
    oss << ",\"level\":\"" << spdlog::level::to_string_view(msg.level).data() << "\"";

    // logger name
    oss << ",\"logger\":\"" << msg.logger_name.data() << "\"";

    // message (JSON escape 간소화 — 큰따옴표, 백슬래시, 줄바꿈만 처리)
    oss << ",\"msg\":\"";
    std::string_view payload(msg.payload.data(), msg.payload.size());
    for (char c : payload) {
        switch (c) {
            case '"': oss << "\\\""; break;
            case '\\': oss << "\\\\"; break;
            case '\n': oss << "\\n"; break;
            case '\r': oss << "\\r"; break;
            case '\t': oss << "\\t"; break;
            default: oss << c; break;
        }
    }
    oss << "\"";

    // source location (있으면)
    if (!msg.source.empty()) {
        oss << ",\"file\":\"" << msg.source.filename << "\""
            << ",\"line\":" << msg.source.line;
    }

    // trace_id: spdlog MDC에서 추출 (thread-local storage)
    // spdlog 1.14+ 에서 spdlog::mdc 지원
    // 여기서는 placeholder — 추후 코어의 trace context와 연동
    // auto trace_id = spdlog::mdc::get("trace_id");
    // if (!trace_id.empty()) {
    //     oss << ",\"trace_id\":\"" << trace_id << "\"";
    // }

    oss << "}";
    return oss.str();
}
```

## 구현 참고사항

### librdkafka API 핵심 함수 정리

| 함수 | 용도 | 스레드 안전성 |
|------|------|-------------|
| `rd_kafka_new()` | Producer/Consumer 핸들 생성 | N/A (초기화) |
| `rd_kafka_produce()` | 메시지 전송 (내부 큐 enqueue) | **스레드세이프** |
| `rd_kafka_poll()` | delivery callback 처리 | Producer 핸들별 직렬 |
| `rd_kafka_flush()` | 큐 비우기 (blocking) | Producer 핸들별 |
| `rd_kafka_consumer_poll()` | 메시지 수신 | Consumer 핸들별 직렬 |
| `rd_kafka_queue_io_event_enable()` | fd 이벤트 연동 | 초기화 1회 |
| `rd_kafka_subscribe()` | 토픽 구독 | Consumer 핸들별 |

### Windows 빌드 주의사항

1. `boost::asio::posix::stream_descriptor`는 Windows 미지원 → `#ifndef _WIN32` 분기
2. `pipe()` 시스템 콜 → Windows 미존재 → `_WIN32` 에서 timer poll fallback
3. librdkafka Windows: winsock2 링크 필요 (vcpkg가 자동 처리)

### 통합 테스트 (Plan 5에서 구현)

이 Plan의 단위 테스트는 브로커 없이 동작하는 범위에 한정한다:
- Producer: `rd_kafka_new()` + `rd_kafka_produce()` (내부 큐 enqueue만)
- Consumer: `rd_kafka_new()` + `rd_kafka_subscribe()` (연결은 비동기)
- Adapter: 라이프사이클 (init → drain → close)
- Sink: spdlog logger → Kafka produce 호출 체인

실제 메시지 라운드트립 (produce → consume)은 docker-compose 기반 통합 테스트에서 검증한다.

### ADR-08 업데이트 (이 Plan 완료 후)

설계서 §5에 명시된 대로, ADR-08의 SPSC 큐 경유 기술을 직접 호출 방식으로 갱신한다:
- **변경 전**: Producer는 SPSC 큐를 경유하여 전용 스레드에서 produce
- **변경 후**: `rd_kafka_produce()` 스레드세이프 특성을 이용, 코어에서 직접 호출. 별도 큐/스레드 불필요.
