## Critical (즉시 수정 필요) — 1건

### C-1. Server::on_accept() — num_cores=0 시 정수 나누기 UB

- **파일**: `server.cpp:208-210`
- **내용**: `% config_.num_cores` — CoreEngine은 0을 `hardware_concurrency()`로 보정하지만, Server는 원본 `config_.num_cores`를 그대로 사용
- **재현**: `Server({.num_cores = 0}).run()` → 첫 accept → `% 0` UB
- **수정**: Server 생성자에서 CoreEngine과 동일하게 0을 보정하거나, `core_engine_->core_count()`로 동기화

## Important — 코드 이슈 (11건)

### I-1. Server::begin_shutdown — active_sessions_ race

- **파일**: `server.cpp:148-190`
- **내용**: `active_sessions_` 0 도달 시점과 코루틴 프레임 해제 타이밍 간 race. `fetch_sub`이 코루틴 co_return 전에 실행되면 프레임 아직 살아있는데 엔진 stop 발생 가능
- **수정**: RAII guard로 `fetch_sub` 시점을 코루틴 함수 최종 종료 직전으로 보장

### I-2. TcpAcceptor::stop() — 백오프 타이머 미취소

- **파일**: `tcp_acceptor.cpp:41-59`
- **내용**: `stop()`이 `acceptor_.close()`만 호출. 백오프 타이머 대기 중이면 타이머 완주 후 코루틴 재개 → TcpAcceptor 소멸 후 `this` 접근 UB
- **수정**: 백오프 타이머를 멤버로 승격 + `stop()`에서 cancel() 호출

### I-3. ErrorSender — null FlatBuffers Offset

- **파일**: `error_sender.cpp:17-25`
- **내용**: `message.empty()` 시 `msg_offset`이 기본 초기화(0) 상태로 전달 → 수신측 `message()->str()`에서 nullptr deref
- **수정**: `builder.CreateString(message.data(), message.size())`로 빈 문자열도 항상 CreateString 처리

### I-4. cross_core_call — T 소멸 스레드 미명시

- **파일**: `cross_core_call.hpp:58-66`
- **내용**: 타임아웃 win 시 result가 target core가 아닌 caller 스레드에서 소멸. RAII 자원 보유 T에서 문제
- **수정**: requires절 또는 API 주석에 "R must be safely destructible from any thread" 명시

### I-5. Server::stop() — 주석 "signal handler 호출 가능" 오해 유발

- **파일**: `server.hpp:103-106`
- **내용**: async-signal-safe하지 않은 코드이나 "signal handler에서 호출 가능"으로 문서화
- **수정**: "signal_set completion handler에서 호출 가능. raw POSIX signal handler에서는 불가"로 수정

### I-6. Server::add_service — Args 값 복사 캡처

- **파일**: `server.hpp:73-83`
- **내용**: `[args...]` 람다 캡처로 move-only Args 전달 불가
- **수정**: `add_service_factory` 사용 안내 주석 추가

### I-7. MpscQueue — head_ 캐시라인 공유

- **파일**: `mpsc_queue.hpp:71-77`
- **내용**: `head_`(producer write)와 `slots_`/`capacity_`/`mask_`(read-only)이 같은 캐시라인. CAS 집중 시 불필요한 캐시 invalidation
- **수정**: `head_`를 `alignas(64)` 전용 라인으로 분리, 불변 필드도 별도 라인

### I-8. TimingWheel — 콜백 예외 무음 삼킴

- **파일**: `timing_wheel.cpp:149-153`
- **내용**: `catch(...)` 블록에서 로깅 없이 예외 삼킴 → 운영 환경 디버깅 불가
- **수정**: spdlog 또는 stderr 로깅 추가

### I-9. TimingWheel — entries_ OOB 방어 부재

- **파일**: `timing_wheel.cpp:61-68`
- **내용**: `free_ids_`에서 꺼낸 id가 `entries_.size()` 내에 있다는 불변식에 assert 없음
- **수정**: `assert(id < entries_.size())` 추가

### I-10. RingBuffer — linear_buf_ raw pointer

- **파일**: `ring_buffer.cpp:83-91`, `ring_buffer.hpp`
- **내용**: `malloc`/`realloc`/`free` 직접 관리 → 예외 안전성 미흡
- **수정**: `std::unique_ptr<uint8_t[], decltype(&std::free)>` 변환 또는 현 상태 유지 + RAII 주석

### I-11. SessionManager — timeout_callback_ 재진입 계약 미문서화

- **파일**: `session_manager.hpp`, `session_manager.cpp:89-111`
- **내용**: 콜백 내에서 `tick()` 호출 시 이터레이터 무효화 위험. 계약이 문서화되어 있지 않음
- **수정**: 헤더 주석에 "콜백 내 tick() 재진입 금지" 명시
