# v0.2.1 코드 리뷰 수정 + 코루틴 전환 설계서

**일자:** 2026-03-06
**목표:** v0.2.0 코드 리뷰 15건 전체 수정 + 전면 코루틴(방안 A) 전환
**선행 문서:** `docs/review/v0.2.0_review.md`

---

## 설계 결정

### 방안 A 채택: 전면 코루틴

- `send()`/`send_raw()` → `awaitable<bool>` 코루틴
- `dispatch()` → `awaitable<expected<void, DispatchError>>` 코루틴
- `ServiceBase::handle()/route()` 핸들러 → `awaitable<void>` 코루틴
- `Server::do_read()` 재귀 콜백 → `read_loop()` while 코루틴
- `TcpAcceptor::do_accept()` 재귀 콜백 → `accept_loop()` while 코루틴

**채택 근거:** 설계 문서(`design-decisions.md`)에 명시된 코루틴 기반 아키텍처와 일관성 유지. 핸들러까지 코루틴으로 통일하여 "동기/비동기 혼용" 문제를 근본적으로 제거.

**기각된 방안 B:** send API는 동기 유지 + 내부 송신 큐 + write drain 코루틴. 핸들러 API 변경 없이 호환 가능하지만 일관성 부족.

## 인터페이스 변경 스펙

### Session (session.hpp)

```cpp
#pragma once

#include <apex/core/frame_codec.hpp>
#include <apex/core/ring_buffer.hpp>
#include <apex/core/wire_header.hpp>

#include <boost/asio/ip/tcp.hpp>
#include <boost/asio/awaitable.hpp>

#include <cstdint>
#include <memory>

namespace apex::core {

using SessionId = uint64_t;

class Session : public std::enable_shared_from_this<Session> {
public:
    enum class State : uint8_t {
        Connected,
        Active,
        // Closing 제거 (I-2): 미사용 상태였음. graceful shutdown은 Closed로 직접 전이.
        Closed,
    };

    Session(SessionId id, boost::asio::ip::tcp::socket socket,
            uint32_t core_id, size_t recv_buf_capacity = 8192);
    ~Session();

    Session(const Session&) = delete;
    Session& operator=(const Session&) = delete;

    /// 프레임 응답을 이 세션에 비동기 전송.
    [[nodiscard]] boost::asio::awaitable<bool>
    async_send(const WireHeader& header, std::span<const uint8_t> payload);

    /// 미리 빌드된 로우 프레임 비동기 전송.
    [[nodiscard]] boost::asio::awaitable<bool>
    async_send_raw(std::span<const uint8_t> data);

    void close();

    [[nodiscard]] SessionId id() const noexcept { return id_; }
    [[nodiscard]] uint32_t core_id() const noexcept { return core_id_; }
    [[nodiscard]] State state() const noexcept { return state_; }
    [[nodiscard]] bool is_open() const noexcept {
        return state_ == State::Connected || state_ == State::Active;
    }
    [[nodiscard]] boost::asio::ip::tcp::socket& socket() noexcept { return socket_; }
    [[nodiscard]] RingBuffer& recv_buffer() noexcept { return recv_buf_; }

    void set_state(State s) noexcept { state_ = s; }

private:
    SessionId id_;
    uint32_t core_id_;
    State state_{State::Connected};
    boost::asio::ip::tcp::socket socket_;
    RingBuffer recv_buf_;
};

using SessionPtr = std::shared_ptr<Session>;

} // namespace apex::core
```

### MessageDispatcher (message_dispatcher.hpp)

```cpp
#pragma once

#include <apex/core/session.hpp>

#include <boost/asio/awaitable.hpp>

#include <array>
#include <cstdint>
#include <expected>
#include <functional>
#include <memory>
#include <span>

namespace apex::core {

enum class DispatchError : uint8_t {
    UnknownMessage,
    HandlerFailed,
};

class MessageDispatcher {
public:
    // 핸들러가 코루틴을 반환 (방안 A 핵심)
    using Handler = std::function<
        boost::asio::awaitable<void>(SessionPtr, uint16_t, std::span<const uint8_t>)>;

    MessageDispatcher() = default;

    void register_handler(uint16_t msg_id, Handler handler);
    void unregister_handler(uint16_t msg_id);

    // dispatch도 코루틴 (내부에서 co_await handler 호출)
    [[nodiscard]] boost::asio::awaitable<std::expected<void, DispatchError>>
    dispatch(SessionPtr session, uint16_t msg_id, std::span<const uint8_t> payload) const;

    [[nodiscard]] bool has_handler(uint16_t msg_id) const noexcept;
    [[nodiscard]] size_t handler_count() const noexcept;

private:
    std::unique_ptr<std::array<Handler, 65536>> handlers_
        = std::make_unique<std::array<Handler, 65536>>();
    size_t handler_count_{0};
};

} // namespace apex::core
```

### ServiceBase (service_base.hpp) — handle/route 변경

```cpp
protected:
    /// 로우 핸들러 등록 (코루틴).
    void handle(uint16_t msg_id,
                boost::asio::awaitable<void> (Derived::*method)(
                    SessionPtr, uint16_t, std::span<const uint8_t>))
    {
        auto* self = static_cast<Derived*>(this);
        dispatcher_->register_handler(msg_id,
            [self, method](SessionPtr session, uint16_t id,
                           std::span<const uint8_t> payload)
                -> boost::asio::awaitable<void> {
                co_await (self->*method)(session, id, payload);
            });
    }

    /// FlatBuffers 타입 핸들러 등록 (코루틴).
    template <typename FbsType>
    void route(uint16_t msg_id,
               boost::asio::awaitable<void> (Derived::*method)(
                   SessionPtr, uint16_t, const FbsType*))
    {
        auto* self = static_cast<Derived*>(this);
        dispatcher_->register_handler(msg_id,
            [self, method](SessionPtr session, uint16_t id,
                           std::span<const uint8_t> payload)
                -> boost::asio::awaitable<void> {
                flatbuffers::Verifier verifier(payload.data(), payload.size());
                if (!verifier.VerifyBuffer<FbsType>()) co_return;
                auto* msg = flatbuffers::GetRoot<FbsType>(payload.data());
                co_await (self->*method)(session, id, msg);
            });
    }
```

## 테스트 패턴

### 코루틴 단위 테스트 헬퍼

```cpp
// 테스트에서 awaitable을 동기적으로 실행하는 헬퍼
template <typename T>
T run_coro(boost::asio::io_context& ctx, boost::asio::awaitable<T> aw) {
    auto future = boost::asio::co_spawn(ctx, std::move(aw), boost::asio::use_future);
    ctx.run();
    ctx.restart();
    return future.get();
}

// void 특수화
inline void run_coro(boost::asio::io_context& ctx, boost::asio::awaitable<void> aw) {
    auto future = boost::asio::co_spawn(ctx, std::move(aw), boost::asio::use_future);
    ctx.run();
    ctx.restart();
    future.get();
}
```

## 병렬 에이전트 구성 (6개)

모든 에이전트에 완전한 인터페이스 스펙을 프롬프트에 제공하여 **파일 충돌 없이** 동시 실행.

### Agent 1: TimingWheel + SessionManager

**수정 이슈:** C-2, C-6, M-1, M-3
**수정 파일:**
- `core/include/apex/core/timing_wheel.hpp`
- `core/src/timing_wheel.cpp`
- `core/src/session_manager.cpp`

**C-2 수정:** `tick()` Phase 3에서 `entries_[id]=nullptr` + `free_ids_.push_back(id)`를 콜백 전으로 이동.
**C-6 수정:** `on_timer_expire()`에서 `sessions_.erase(session_it)`를 콜백 전으로 이동.
**M-1 수정:** `active_count_` 멤버 변수 추가, schedule/cancel/tick에서 증감.
**M-3 수정:** `schedule()` 헤더 주석에 "ticks_from_now=0이면 다음 tick()에서 즉시 만료" 명시.

### Agent 2: Session 코루틴

**수정 이슈:** C-1, I-2
**수정 파일:**
- `core/include/apex/core/session.hpp`
- `core/src/session.cpp`

**C-1 수정:** `send()`/`send_raw()` 제거, `async_send()`/`async_send_raw()` 코루틴 추가.
- 내부: `co_await boost::asio::async_write(socket_, buffer, as_tuple(use_awaitable))`
**I-2 수정:** `State::Closing` enum 제거. 주석의 생명주기를 `Connected -> Active -> Closed`로 변경.

### Agent 3: MessageDispatcher + ServiceBase 코루틴

**수정 이슈:** Handler 코루틴화, I-1
**수정 파일:**
- `core/include/apex/core/message_dispatcher.hpp`
- `core/src/message_dispatcher.cpp`
- `core/include/apex/core/service_base.hpp`

**Handler 변경:** `std::function<void(...)>` → `std::function<awaitable<void>(...)>`
**dispatch 변경:** `co_await handler(...)` + try-catch 유지.
**I-1 수정:** `bind_dispatcher()`에 `assert(!started_)` 추가.

### Agent 4: Server + TcpAcceptor 코루틴

**수정 이슈:** C-3, C-4, C-5, I-5, I-6, M-2
**수정 파일:**
- `core/include/apex/core/server.hpp`
- `core/src/server.cpp`
- `core/include/apex/core/tcp_acceptor.hpp`
- `core/src/tcp_acceptor.cpp`

**Server 변경:**
- `do_read()` / `start_read()` 제거 → `read_loop()` 코루틴
- `process_frames()` → 코루틴 (`co_await dispatcher_.dispatch(...)`)
- `on_accept()` 내에서 `co_spawn(io_ctx_, read_loop(session), detached)`
- C-4: `co_await process_frames()` 후 `if (!session->is_open()) break;`
- C-5: `copy_to_recv_buf` 실패 시 `break` (→ remove_session)
- I-5: `stop()` 내부를 `post(io_ctx_, ...)` 로 감싸기

**TcpAcceptor 변경:**
- `do_accept()` 제거 → `accept_loop()` 코루틴
- `start()` → `co_spawn(io_ctx_, accept_loop(), detached)`
- C-3: 코루틴 프레임이 수명 관리, `[this]` 캡처 문제 해소
- I-6: 생성자에 `boost::asio::ip::tcp protocol` 파라미터 추가

### Agent 5: 기타 독립 수정

**수정 이슈:** I-3, I-4
**수정 파일:**

**I-4 수정:** `ErrorSender::build_error_frame` 헤더 주석에 "message가 비어있으면 FlatBuffers 필드는 null (수신측에서 nullptr 체크 필요)" 명시.

## 빌드 참고

- Windows 10, VS2022, MSVC 19.44
- C++20 코루틴 지원 확인 필요: `/std:c++latest` 또는 `/std:c++20` + `/await:strict` 불필요 (MSVC 19.44는 C++20 코루틴 기본 지원)
- Boost.Asio `awaitable<T>` 사용: `#include <boost/asio/awaitable.hpp>`, `#include <boost/asio/co_spawn.hpp>`, `#include <boost/asio/use_awaitable.hpp>`
- `as_tuple(use_awaitable)`: `#include <boost/asio/as_tuple.hpp>` — 예외 대신 `[ec, result]` 튜플 반환
- vcpkg Boost 버전이 코루틴 지원하는지 확인 (Boost 1.80+ 권장)
- CMakeLists.txt에 코루틴 관련 추가 링크 불필요 (헤더 온리)
