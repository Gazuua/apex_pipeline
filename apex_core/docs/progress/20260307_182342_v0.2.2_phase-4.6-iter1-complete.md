### FIX-2: ProtocolBase CRTP 추상화
- `core/include/apex/core/protocol_base.hpp` 신규 생성
  - CRTP 기반 프로토콜 추상화 (try_decode / consume_frame)

- `core/include/apex/core/tcp_binary_protocol.hpp` 신규 생성
  - FrameCodec 래핑, ProtocolBase 구현체
- `core/src/server.cpp`: `FrameCodec::*` → `TcpBinaryProtocol::*` 전환
- `core/tests/unit/test_tcp_binary_protocol.cpp` 신규 (4 테스트)

### FIX-3: 에러 전파 파이프라인
- `core/include/apex/core/message_dispatcher.hpp`: 핸들러 반환 `awaitable<void>` → `awaitable<Result<void>>`
- `core/include/apex/core/service_base.hpp`: handle/route 핸들러 시그니처 동일 전환
  - `route<T>()`: FlatBuffers 검증 실패 시 `ErrorCode::FlatBuffersVerifyFailed` 반환
- `core/src/server.cpp`: DispatchError + ErrorCode 이중 에러 체크 및 ErrorResponse 전송
- 영향받는 모든 파일 시그니처 통일:
  - 단위 테스트: test_message_dispatcher, test_service_base, test_flatbuffers_dispatch
  - 통합 테스트: test_server_e2e, test_pipeline_integration
  - 예제: echo_server, chat_server
